# 품질 속성 시나리오 (QAS) — Variant 2: NFR 항목별 1:1 매핑

> **본 문서는 [07-QAS.md](./07-QAS.md)의 비교용 버전이다.**
> - [07-QAS.md](./07-QAS.md): 중요도(`[H,H]`/`[H,M]`) 기반 선별 → 15개
> - **본 문서**: `05-NFR.md`의 모든 NFR 항목에 대해 1:1로 핵심 QAS 1개씩 도출 → 16개
>
> **동기화 규칙**: [05-NFR.md](./05-NFR.md) 변경 시 본 문서도 함께 갱신해야 한다. ([CLAUDE.md 7항](../CLAUDE.md))
> - NFR 추가/삭제 → QAS 추가/삭제
> - 임계값·측정 구간 변경 → 대응 QAS의 **Response Measure** 동기화
>
> 품질 속성 분류는 **ISO/IEC 25010:2023** 기준이며, 우선순위는 `[Business Value, Technical Risk]`(H/M/L)로 표기한다.

## 1. Utility Tree

본 문서의 도출 원칙은 "NFR 항목별 1개"이므로, Utility Tree의 모든 리프 노드가 QAS로 도출된다.

```
시스템 품질 (Utility)
│
├─ Performance Efficiency
│  ├─ Time Behaviour
│  │  ├─ Intent 인지 알림 ≤ 2초                         [H, H] → NFR-IDS-01 / QAS-001
│  │  ├─ Routing 판별 ≤ 1초                              [M, M] → NFR-IDS-02 / QAS-002
│  │  ├─ IDS 자체 Task 처리 ≤ 5초                        [M, M] → NFR-IDS-03 / QAS-003
│  │  ├─ Communication Latency                          [H, H] → NFR-MAF-04 / QAS-009
│  │  └─ 사용자 개입 반영 ≤ 2초                          [H, M] → NFR-MAF-05 / QAS-010
│  ├─ Resource Utilisation
│  │  ├─ IDS 자원 사용량                                 [H, M] → NFR-IDS-04 / QAS-004
│  │  └─ MAF 자원 사용량                                 [H, M] → NFR-MAF-06 / QAS-011
│  └─ Capacity / Scalability
│     └─ N-party 자원 선형 확장                          [H, H] → NFR-MAF-09 / QAS-014
│
├─ Reliability
│  ├─ Maturity
│  │  ├─ IDS 자체 Task 성공률 ≥ 90%                      [H, M] → NFR-IDS-05 / QAS-005
│  │  └─ Communication Task 성공률 ≥ 95%                [H, H] → NFR-MAF-02 / QAS-007
│  ├─ Fault Tolerance
│  │  └─ 일시적 오류 자동 복구 ≥ 99%                     [H, H] → NFR-MAF-01 / QAS-006
│  ├─ Recoverability
│  │  └─ 세션 복구 성공률 ≥ 95%                          [H, H] → NFR-MAF-03 / QAS-008
│  └─ Consistency
│     ├─ 다자간 상태 일관성 0%                           [H, H] → NFR-MAF-10 / QAS-015
│     └─ 다자간 합의 결과 일관성 0%                      [H, H] → NFR-MAF-11 / QAS-016
│
├─ Security
│  └─ Confidentiality
│     └─ TLS 1.3 + Private Knowledge 외부 전송 0건       [H, H] → NFR-MAF-07 / QAS-012
│
└─ Maintainability (Traceability)
   └─ Communication 이력 추적 100% / 조회 ≤ 2초          [M, H] → NFR-MAF-08 / QAS-013
```

## 2. 품질 속성 시나리오 (QAS)

### QAS-001 Intent 인지 알림 2초 이내 표시
- **Source**: 사용자 (End User)
- **Stimulus**: 사용자 입력이 IDS에 전달됨 (음성·텍스트 입력 모듈 처리 완료 시점)
- **Artifact**: IDS Intent 인지 모듈, UI 알림 컴포넌트
- **Environment**: 정상 운영, 온디바이스 LLM 정상 로딩
- **Response**: Intent 인지 후 "처리 중" 포함 UI 알림이 표시됨
- **Response Measure**: 알림 표시까지 **2초 이내**
- **Mapped NFR**: NFR-IDS-01
- **Mapped QA leaf**: Performance Efficiency — Time Behaviour (Intent 인지 알림) [H, H]

---

### QAS-002 IDS Routing 처리 주체 판별 1초 이내
- **Source**: Intent 인지가 완료된 IDS 내부 흐름
- **Stimulus**: Routing 모듈이 Intent를 수신하여 처리 주체(IDS 자체 / MAF) 판별을 시작함
- **Artifact**: IDS Routing 모듈 (온디바이스 경량 LLM 분류기)
- **Environment**: 정상 운영, 온디바이스 LLM 정상 로딩
- **Response**: 처리 주체 판별이 완료됨
- **Response Measure**: 판별 시작 ~ 판별 완료 **1초 이내**
- **Mapped NFR**: NFR-IDS-02
- **Mapped QA leaf**: Performance Efficiency — Time Behaviour (Routing) [M, M]

---

### QAS-003 IDS 자체 Task 처리 5초 이내 완료
- **Source**: IDS Routing 모듈에서 "자체 처리"로 분류된 Task
- **Stimulus**: IDS Task 처리 모듈이 단순 로컬 명령·정보 검색 수준의 Task를 수신함
- **Artifact**: IDS Task 처리 모듈, 로컬 명령 실행기, 정보 검색 어댑터
- **Environment**: 정상 운영, 온디바이스 자원 정상
- **Response**: 처리 결과가 반환됨
- **Response Measure**: 판별 완료 ~ 결과 반환 **5초 이내**
- **Mapped NFR**: NFR-IDS-03
- **Mapped QA leaf**: Performance Efficiency — Time Behaviour (IDS Task) [M, M]

---

### QAS-004 IDS 온디바이스 자원 사용량 준수
- **Source**: 동시 동작 포그라운드 앱, OS 자원 관리자
- **Stimulus**: IDS가 백그라운드 Intent 인지 동작을 수행함
- **Artifact**: IDS 백그라운드 서비스, 온디바이스 LLM 추론 엔진
- **Environment**: Galaxy S26(Android, RAM 12~16GB, NPU 탑재), 포그라운드 앱 활성
- **Response**: 자원 사용이 임계값 내에서 유지되어 포그라운드 앱 성능에 영향이 없음
- **Response Measure**: CPU 평균 **10%** / 피크 **25%** / NPU **70%** / Memory **200MB** 이하
- **Mapped NFR**: NFR-IDS-04
- **Mapped QA leaf**: Performance Efficiency — Resource Utilisation (IDS) [H, M]

---

### QAS-005 IDS 자체 Task 처리 성공률 90% 이상
- **Source**: IDS가 자체 처리하기로 분류한 Task 모집단
- **Stimulus**: 자체 처리 Task가 실행됨
- **Artifact**: IDS Task 처리 모듈 및 그 하위 의존 컴포넌트
- **Environment**: 정상 운영
- **Response**: 기술적 오류(크래시·타임아웃·예외 미처리) 없이 결과가 반환됨 (의도 해석 정확도는 별도 평가)
- **Response Measure**: 자체 Task 처리 성공률 **90% 이상**
- **Mapped NFR**: NFR-IDS-05
- **Mapped QA leaf**: Reliability — Maturity (IDS 자체 처리) [H, M]

---

### QAS-006 일시적 오류 자동 복구율 99% 이상
- **Source**: 네트워크·메시지 계층의 일시적 장애
- **Stimulus**: Communication 진행 중 패킷 손실·일시적 연결 불안정·메시지 처리 실패 발생
- **Artifact**: 메시지 큐, 재전송 로직, 세션 상태 관리자
- **Environment**: 일시적 네트워크 불안정 (완전 단절은 NFR-MAF-03 범위로 제외)
- **Response**: 사용자 인지 없이 자동 재전송·재시도가 수행되어 세션이 중단 없이 정상 완료됨
- **Response Measure**: 일시적 오류 발생 케이스 중 자동 복구 완료율 **99% 이상**
- **Mapped NFR**: NFR-MAF-01
- **Mapped QA leaf**: Reliability — Fault Tolerance [H, H]

---

### QAS-007 Communication Task 성공률 95% 이상
- **Source**: MAF에 위임된 Communication Task 전체 모집단
- **Stimulus**: 다양한 케이스(Negotiation·Collaboration·Knowledge Sharing·Remote Monitoring)의 Task가 시작됨
- **Artifact**: MAF 전체(Meta/Sub/Orchestrator, 통신 채널, Action 모듈)
- **Environment**: 정상 운영 ~ 일시적 장애 혼재
- **Response**: Task가 케이스별 정상 흐름(합의 타결 또는 합의 불가 판정 포함)으로 종결됨
- **Response Measure**: 전체 시도 대비 기술적 Task 성공률 **95% 이상** (크래시·타임아웃·예외 미처리만 실패로 집계)
- **Mapped NFR**: NFR-MAF-02
- **Mapped QA leaf**: Reliability — Maturity (전체 Task 성공률) [H, H]

---

### QAS-008 세션 복구 성공률 95% 이상
- **Source**: 앱 강제 종료, 네트워크 완전 단절, OS 프로세스 재시작
- **Stimulus**: 진행 중이던 Communication 세션이 중단됨
- **Artifact**: 세션 상태 영속화 저장소, 세션 복구 매니저, 통신 채널 재수립 모듈
- **Environment**: 단절 후 연결 복구, 세션 상태 저장 정보 유효
- **Response**: 중단 시점부터 세션이 정상 재개됨
- **Response Measure**: 복구 시도 케이스 중 정상 재개율 **95% 이상** (우선 적용: Negotiation)
- **Mapped NFR**: NFR-MAF-03
- **Mapped QA leaf**: Reliability — Recoverability [H, H]

---

### QAS-009 Communication Latency 케이스별 임계값 준수
- **Source**: MAF에 Task를 위임한 사용자/IDS
- **Stimulus**: Communication Task가 MAF에 전달되어 처리됨
- **Artifact**: Meta/Sub-Agent, Orchestrator, 통신 채널, Action 수행 모듈
- **Environment**: 정상 네트워크, 1:1 또는 N-party 협상
- **Response**: 케이스별 임계값 내에서 Action 완료까지 종결됨
- **Response Measure**:
  - 총 소요(Task 전달 ~ Action 완료): **180초 이내**
  - Discovery: **5초 이내**
  - Negotiation 2인: **60초 이내**, N-party(N≥3): **60 + (N-2)·30초 이내**
  - Collaboration: **10초 이내**, Knowledge Sharing: **10초 이내**
  - Remote Monitoring 알림: **5초 이내**
- **Mapped NFR**: NFR-MAF-04
- **Mapped QA leaf**: Performance Efficiency — Time Behaviour (Communication Latency) [H, H]

---

### QAS-010 사용자 개입 시 2초 이내 반영
- **Source**: 사용자 (End User)
- **Stimulus**: 진행 중인 Communication에 대해 UI로 직접 개입(취소·수정 등) 수행
- **Artifact**: UI, IDS 변경 인지 경로, MAF Intent 적용 모듈
- **Environment**: 정상 운영, Communication 세션 진행 중
- **Response**: MAF가 변경 Intent를 인지·적용 완료함
- **Response Measure**: UI 조작 ~ MAF 적용 완료 **2초 이내** (상대 에이전트 통보 등 후속 처리는 제외)
- **Mapped NFR**: NFR-MAF-05
- **Mapped QA leaf**: Performance Efficiency — Time Behaviour (사용자 개입 반영) [H, M]

---

### QAS-011 MAF 온디바이스 자원 사용량 준수 (1:1 기준)
- **Source**: 동시 동작 포그라운드 앱, OS 자원 관리자
- **Stimulus**: MAF가 1:1 Communication을 수행함
- **Artifact**: Meta/Sub-Agent, 온디바이스 LLM 추론 엔진, Multi-Agent 상태 관리자
- **Environment**: Galaxy S26 기준, 포그라운드 앱 활성, 1:1(2인) Communication
- **Response**: 자원 사용이 임계값 내에서 유지됨 (N-party는 NFR-MAF-09를 따름)
- **Response Measure**: CPU 평균 **15%** / 피크 **40%** / NPU **70%** / Memory **300MB** 이하
- **Mapped NFR**: NFR-MAF-06
- **Mapped QA leaf**: Performance Efficiency — Resource Utilisation (MAF) [H, M]

---

### QAS-012 에이전트 간 통신 개인정보 보안
- **Source**: 외부 에이전트, 잠재적 네트워크 도청자
- **Stimulus**: 에이전트 간 Communication을 위해 데이터 전송이 발생함
- **Artifact**: 통신 채널(TLS), 지식 보안 등급 분류기(FR-MAF-05), 전송 필터
- **Environment**: 모든 Communication 케이스
- **Response**: 전송 계층 암호화되고 Private Knowledge는 외부로 전송되지 않음
- **Response Measure**: TLS **1.3 이상**, Private Knowledge 외부 전송 **0건**
- **Mapped NFR**: NFR-MAF-07
- **Mapped QA leaf**: Security — Confidentiality [H, H]

---

### QAS-013 Communication 이력 추적 완전성·조회 응답성
- **Source**: 사용자, 분쟁 조정자, 디버깅 담당자
- **Stimulus**: Communication 상태 전이 이벤트 발생, 또는 사용자 이력 조회
- **Artifact**: 이벤트 로깅 모듈, 이력 저장소, 조회 UI(FR-MAF-07)
- **Environment**: 모든 Communication 케이스, 정상 운영
- **Response**: 모든 상태 전이가 누락 없이 기록되고 조회 시 즉시 응답함
- **Response Measure**: 이벤트 로그 기록률 **100%**, 조회 응답 시간 **2초 이내**
- **Mapped NFR**: NFR-MAF-08
- **Mapped QA leaf**: Maintainability — Traceability [M, H]

---

### QAS-014 N-Party 참여자 증가 시 자원 선형 확장
- **Source**: N-party Negotiation 시작 요청
- **Stimulus**: 참여자 수가 1명 증가함 (기준값: 1:1, CPU 피크 40% / Memory 300MB)
- **Artifact**: Meta Agent, 메시지 파서·라우터, 협상 상태 저장소, NPU 추론 엔진
- **Environment**: Galaxy S26 기준, 다자간(N≥3) 협상 진행
- **Response**: 추가 참여자에 따른 자원 사용 증가가 선형(O(N)) 범위 내로 제한됨
- **Response Measure**:
  - CPU 피크: 참여자 1명당 **2% 이내** (3명→42%, 5명→46%, 7명→50% 이하)
  - Memory: 참여자 1명당 **20MB 이내** (3명→320MB, 5명→360MB, 7명→400MB 이하)
- **Mapped NFR**: NFR-MAF-09
- **Mapped QA leaf**: Performance Efficiency — Capacity / Scalability [H, H]

---

### QAS-015 다자간 협상 상태 일관성 100%
- **Source**: 다자간 협상에 참여한 모든 에이전트
- **Stimulus**: 협상 라운드 진행 중 한 참여자가 상태 변경(제안·역제안·수락·거절·철회)을 발생시킴
- **Artifact**: 상태 동기화 프로토콜, 메시지 브로드캐스트 모듈, 참여자별 상태 저장소
- **Environment**: 다자간(N≥3) 협상 진행 중, 정상 네트워크
- **Response**: 모든 참여 에이전트가 동일한 "현재 협상 상태"를 공유함
- **Response Measure**: 상태 불일치 발생률 **0%**, 동기화 지연 **2초 이내**. 불일치 시 해당 에이전트의 진행은 일시 중단.
- **Mapped NFR**: NFR-MAF-10
- **Mapped QA leaf**: Reliability — Consistency (State Consistency) [H, H]

---

### QAS-016 다자간 합의 결과 일관성 100%
- **Source**: 다자간 협상 완료 시점의 모든 참여 에이전트
- **Stimulus**: N-party Negotiation이 합의 타결로 종결됨
- **Artifact**: 합의 결과 브로드캐스트 모듈, ACK 수집기, 후속 Action 트리거(FR-MAF-09)
- **Environment**: 다자간(N≥3) 협상 타결 직후
- **Response**: 모든 참여자가 동일한 최종 합의 결과를 수신·확인하고, 모든 ACK 완료 후에만 후속 Action이 트리거됨
- **Response Measure**: 합의 결과 불일치 발생률 **0%**, 합의 완료 기준은 **모든 참여자의 ACK 수신**
- **Mapped NFR**: NFR-MAF-11
- **Mapped QA leaf**: Reliability — Consistency (Consensus Consistency) [H, H]

## 3. 07-QAS.md와의 차이점

| 항목 | 07-QAS.md | 07-QAS-2.md (본 문서) |
|---|---|---|
| 도출 원칙 | 중요도(`[H,H]`/`[H,M]`) 기반 선별 | NFR 항목별 1:1 매핑 |
| QAS 개수 | 15개 | 16개 |
| 포함되는 `[M,M]` NFR | 제외 (3절에 사유 명시) | QAS-002(NFR-IDS-02), QAS-003(NFR-IDS-03) 포함 |
| Communication Latency 분해 | 3개 QAS로 분해 (총소요/Remote Monitoring/N-party) | 1개 QAS로 통합 (QAS-009) |

| 장점 | 우선순위 가시성, 검증 집중도 ↑ | NFR 추적성 1:1, 누락 위험 ↓ |
| 단점 | `[M,M]` 항목 별도 관리 필요 | 같은 NFR의 다중 측정항목이 한 QAS에 묶여 가독성 ↓ |
