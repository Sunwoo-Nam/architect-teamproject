# 제약 사항 명세서 (Constraints Specification)


> Constraint는 아키텍처 설계 시 협상하거나 완화할 수 없는 **비협상 한계(Hard Limit)**이다.

> NFR이 "어느 수준을 달성해야 하는가"를 정의한다면, Constraint는 "이 경계를 넘는 설계는 선택지에서 제외된다"를 정의한다.

> 각 항목은 이유 없이 완화될 수 없으며, 변경 시 이해관계자 전원의 명시적 승인이 필요하다.


---


## 1. 기술·플랫폼 제약


### CON-01. 실행 플랫폼 Android 한정


정의: 본 시스템은 Android 운영체제 기반 스마트폰에서만 동작해야 한다.


상세 설명:

* iOS, HarmonyOS 등 타 OS는 현재 범위에서 제외된다.

* Android API 수준의 최소 버전은 아키텍처 설계 단계에서 별도로 확정한다.


---


### CON-02. 상대방 기기 동일 MAF 탑재 전제


정의: 협상의 상대방 기기는 본 시스템과 동일한 MAF(Multi-Agent Framework)가 탑재된 기기임을 전제한다.


상세 설명:

* 이기종 에이전트 구현(타사 Agent 프레임워크)과의 상호운용성은 현재 범위에서 제외된다.

* 협상 메시지 포맷·상태 머신을 양측이 동일하게 공유함을 가정할 수 있다.
