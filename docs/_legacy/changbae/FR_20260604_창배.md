# 기능 요구사항


## 1. IDS 기능 요구사항


### FR-IDS-01. 다중 컨텍스트 기반 의도 추출
사용자의 발화, 현재 화면 상태(Screen Context), 디바이스 상태(GPS, 배터리, 네트워크 연결), 생체 리듬(Wearable Data) 등을 주기적으로 수집하여 비명시적 의도를 도출해야 한다.

리뷰1. Intent는 2종류: 사용자의 직접 명령, 주기적으로 수집해서 비명시적 의도 도출


### FR-IDS-02. 로컬 처리 가능 여부 판별 및 라우팅
도출한 의도를 로컬 엔진(ZeroClaw)로 처리 가능한 태스크인지 MAF로 전달해야 할 태스크인지를 식별하여 라우팅 해야 한다.


### FR-IDS-03. 새로운 의도 인식 및 동적 컨텍스트 맵핑
사용자가 기존에 지시한 내용과 충돌하거나 완전히 새로운 의도를 발화할 경우, 이를 이전 의도의 맥락과 대조하여 로컬 엔진 또는 MAF로 전달하여야 한다.


## 2. Multi-Agent Framework(MAF) 기능 요구사항


### FR-MAF-01. Discovery 기능
협상이나 협업을 수행할 대상 에이전트의 엔드포인트를 탐색하고 상호 인증을 통해 통신 가능한 상태를 수립해야 한다.


### FR-MAF-02. 1:1 자율 협상(Negotiation) 기능
단일 상대방 에이전트와 일정 조율, 조건 합의 등의 협상(Negotiation)을 자율적으로 수행해야 한다.


### FR-MAF-03. N-Party 협상 기능
3명 이상의 에이전트가 참여하는 환경에서, 무한 루프나 교착 상태에 빠지지 않으며 다자간 합의 프로토콜을 구동하여 협상을 수행해야 한다.


### FR-MAF-04. Collaboration(공동 문서 작업) 기능
에이전트 간 공동 문서 작성 등의 협업 워크플로우를 지원해야 한다.


### FR-MAF-05. Knowledge Sharing 기능
각 사용자의 에이전트가 보유한 지식(Knowledge)를 타 에이전트와 공유할 수 있어야 한다.

리뷰1. Knowledge Sharing의 예시가 뭐가 있지? Negotiation의 하위 로직으로 분류되지 않는 별도의 시나리오가 있나?

### FR-MAF-06. Knowledge Sharing - Privacy 격리 기능
Knowledge Sharing 시 외부로 공유 가능한 데이터(Shared)와 공유할 수 없는 민감 데이터(Private)를 격리하여야 한다.


### FR-MAF-07. Remote Monitoring 기능
보호자/피보호자(Parent/Child) 등으로 이루어진 에이전트 간에, 상대방에 대한 권한 위임을 바탕으로 상대방 단말의 특정 상태, 알림, 데이터 등을 원격으로 수집하고 모니터링 할 수 있는 기능을 제공해야 한다.

리뷰1. Child care의 경우에는 기존 안드로이드 기능에서도 부모가 제어할 수 있다.

리뷰2. Parent care의 경우에는 새로운 접근이 가능할 것. 부모의 소유 기기에 대해 자녀의 접근 권한. 


### FR-MAF-08. 협상 상태에 대한 실시간 가시성 제공 기능
백그라운드에서 진행 중인 에이전트 간의 협상, 협업, 공유 상태(현재 진행 턴, 남은 쟁점 등)을 사용자가 원할 때 언제든 UI를 통해 명확히 확인할 수 기능을 제공 해야 한다.


### FR-MAF-09. 협상 중 사용자의 런타임 직접 개입 기능
자율 협상이 진행되는 도중 사용자가 명시적은 UI 조작이나 새로운 의도 발화(IDS 연동)을 통해 협상의 조건 변경, 강제 수락, 진행 취소 등을 지시할 경우, 이를 즉각적으로 반영해야 한다.


### FR-MAF-10 합의안 후속 실행(Tool Calling) 기능
상대방 에이전트와의 협상이나 합의가 최종 완료되었을 때, 도출된 결과를 바탕으로 디바이스 내의 캘린더 등록, 예약 API 호출, 알림 설정 등 실질적인 상태 변경 액션을 실행해야 한다.
