# DP01 — 논의 요약 (재개용)

> **목적**: DP01(N-party 커뮤니케이션) 관련 논의의 *결론·전환점·열린 질문*을 다음 작업 재개 시점에 필요한 만큼만 정리.
> **현재 본문 상태**: [`DP01-N명 커뮤니케이션 시나리오.md`](DP01-N명%20커뮤니케이션%20시나리오.md) — 마지막 커밋 시점(A·B 2-안 비교, 보안성 A/B 모두 ★★☆) 으로 복구됨. 논의된 추가 안(C, RC, A'·B', 매트릭스 재구성 등)은 본문에 반영되어 있지 않음.
> **마지막 논의일**: 2026-06-11

---

## 1. 현재 본문(DP01)에 반영되어 있는 상태

- 결정축: 중앙 집중형(A) vs 탈중앙화(B) 2-안 비교
- 보안성: A·B 모두 ★★☆ — 최소 공개 솔루션(`QS-019`/`NFR-006`)이 두 안의 보안 트레이드오프를 상쇄한다는 결론이 본문에 반영됨
- 자원 효율·정확성·종결 → A 유리, 신뢰성(복구) → B 유리
- 권고: 도메인별 선택(소규모 정형 → A, 대규모 비동기 → B), 하이브리드는 권고 절에서 한 줄 언급만
- drawio: A(좌)·B(우) 두 패널

---

## 2. 논의의 주요 전환점

### 2.1 보안성 평가 정정 — A/B 모두 ★★☆

처음에는 A ★☆☆ / B ★★★ 로 평가했으나, 다음 통찰로 동등 평가로 정정:

- **노출 표면**(내 정보를 보는 상대 단말 수): A=1, B=N-1 → A 유리
- **표적 가치·권력 비대칭**: A는 한 단말에 집중, B는 분산 → B 유리
- 두 차원이 *상충*하므로 토폴로지의 보안 격차는 결정축이 되지 못함
- **최소 공개 솔루션**이 적용되면 공유 정보 자체가 추상화되어 양쪽 위험이 모두 누그러짐

→ 보안 축은 본 DP에서 *결정축에서 빠지고*, 후속 DP에서 "최소 공개 추상화 수준" 합의로 이관.

### 2.2 하이브리드(C) 시도 → 결국 폐기

"평상시 mesh + 확정 시 임시 리더"의 하이브리드 안을 잠시 본문에 추가했으나, 검토 결과:

- 결정론 집계 도메인에서 임시 리더의 역할이 *장식적* — 모두가 같은 결정에 도달하면 "선언자"는 형식적
- C의 정확성 ★★★ 평가가 *과한 평가*였음 — 입력 수집 단계의 mesh 한계가 그대로 남음
- C는 *비결정적 집계*(LLM 기반 후보 생성, 창의적 협상) 도메인에서만 의미가 살아남

→ 본 프로젝트의 결정론 집계 가정 하에서는 C ≈ B 로 수렴. 본문에서 제거.

### 2.3 회전 코디네이터(RC) 시도 → A의 변형으로 판명

"매 라운드 코디네이터를 회전시키면 SPOF를 시간 한정하면서 B의 자원·중복 계산 문제를 해결"이라는 안을 검토했으나:

- 매 순간 단일 권위 + star 토폴로지 → *어느 순간도 A와 동일한 구조*
- 회전은 *시간 차원의 운영 정책*일 뿐, 토폴로지 결정이 아님
- 결국 RC는 본 DP의 결정 사항이 아니라 **A의 후속 DP에서의 운영 정책 옵션**

→ "Coordinator 선출/재선출 정책"(현 8.1) 항목에 흡수되는 방향.

### 2.4 그 외 토폴로지 후보 탐색 — 모두 본 제약에 부적합

토큰 패싱(링), 집계 트리, 사전 공개 디렉토리, 양자 캐스케이드 — 각각 본 프로젝트 제약(모바일·N 작음·세션 짧음·신뢰 모델 양호)과 어딘가에서 충돌:

| 후보 | 결정적 문제 |
|---|---|
| 토큰 패싱 | 순차 처리 = 모바일에서 wall-clock 매우 느림, SPOF 형태만 다름 |
| 집계 트리 | N=3~10 규모에선 과한 복잡도, 부분 SPOF |
| 사전 공개 | 상시 정보 공개 = 보안 NFR과 충돌, 동적 조정 불가 |
| 양자 캐스케이드 | 정보 손실, 결과의 경로 의존성 |

→ 본 프로젝트 제약이 설계 공간을 좁게 만들고, 그 좁은 공간에서 A가 가장 정합적이라는 잠정 결론.

### 2.5 시나리오 검증 — 본문 별점 일부가 흔들렸음

N=4·8의 walk-through 시나리오로 검증해 본 결과:

- **속도(지연)** 평가가 *동기 응답 대기*를 무시하고 *메시지 수*만으로 매겨졌을 가능성: A의 응답 대기 누적이 B의 mesh wall-clock보다 느릴 수 있음
- "결정론 집계 깨짐"의 진짜 원인이 *집계 시점* 자체가 아니라 **"기다림의 끝을 누가 선언하는가"** 라는 *경계 합의* 문제임이 드러남 — 이는 정책으로 사전 정의 가능
- B의 신뢰성 ★★★ 근거가 "SPOF 없음" 하나로 좁혀짐 (split-brain 위험은 정책으로 완화 가능)

→ 본문 평가가 *완전히 틀린 건 아니지만 근거가 좁아졌다*는 인식. 시나리오 부록을 본문에 넣을지 미정.

### 2.6 매트릭스 재구성 시도 → 용어 문제로 보류

결정축을 **"권위 소재 × 정보 소유"** 2축 매트릭스로 분해하고, A'(단일 권위 + 부분 보유)·B'(분산 권위 + 부분 보유) 두 비대각 셀을 추가 비교하려 했음. 본문 전체 재작성까지 완료했으나:

- **"권위(Authority)"** 라는 용어가 결정론 알고리즘 가정과 충돌 — 알고리즘이 결정을 내리는데 "권위를 가진 사람"이 따로 있는 게 아님
- "권위"라는 단어가 *재량적 판단*을 함의해 실체 없는 추상을 가림
- 더 정확한 framing은 **"계산 위치(Computation Locus)"** 또는 **"정보 위치(Information Location)"** 같은 *배치(placement)* 차원

→ 매트릭스 재구성은 *방향성은 살리되* 용어를 정리한 뒤 재시도. 본문은 일단 원복.

## 3. 지금까지 확정된 (잠정적) 결론

1. **결정론적 집계를 전제로 한다** — LLM 추론은 각 단말 로컬, 외부로는 추상화된 결과만. 이는 결정축이 아니라 *전제*.
2. **본 프로젝트 제약 하에서 A가 더 정합적**으로 보임. 단, 본문은 A·B 양안 비교 형태로 둠 (단정하지 않음).
3. **보안성은 결정축에서 제외**. 최소 공개 솔루션이 토폴로지 격차를 상쇄하므로.
4. **세부 알고리즘·연결 방식은 본 DP 범위 밖**. 정족수·결정론 규칙·라우팅·재선출 알고리즘 등은 후속 DP.

## 4. 다음 작업 재개 시 열린 질문

### 4.1 결정축 재정의 (가장 큰 미해결)

매트릭스 재구성을 다시 시도할 때 *용어*를 다음 중 어떤 방향으로 잡을지:

- **계산 위치 × 정보 위치** (단일 실행 vs 중복 실행 × 한 곳 보유 vs 분산 보유)
- **책임 분배 차원** (입력 수집·집계·종결·결과 배포 등 책임 단위로 분해)
- **다른 차원** (논의 필요)

핵심은 "권위" 같은 *재량 함의 용어*를 피하면서 SW 설계 차원의 트레이드오프를 정직하게 드러내는 것.

### 4.2 A의 SPOF 완화 방법 정리

본 DP에서 다루지 않고 후속 DP로 넘긴 항목:
- Coordinator 재선출 정책
- Coordinator 회전 정책 (RC 옵션)
- Coordinator 상태 영속화 시점·범위 (`QS-016` 연계)

이 항목들을 *어느 후속 DP*로 보낼지 명시 필요.

### 4.3 부분 보유(A') 셀의 본격 평가

매트릭스 재구성과 별개로, **단일 실행 + 부분 정보 보유** 셀이 본 프로젝트의 보안 강화 옵션으로 의미 있는지 본격 검토:
- "최소 메타데이터 스키마" 정의의 부담이 어느 정도인지
- 부울/집합 신호만으로 도달 가능한 후보의 *질*이 어느 정도인지
- 도메인별로 채택 여부가 달라지는지

### 4.4 시나리오의 본문 편입 여부

N=4·8 walk-through를 부록으로 본문에 추가할지, 별도 문서로 둘지, 혹은 본문 별점 평가의 근거 강화용으로만 활용할지.

### 4.5 권위 부활 엣지 케이스

결정론 알고리즘 가정이 완전치 않은 경우 — 타이브레이크 동점, LLM 해석 모호성, 정책 미커버 엣지 케이스 — 에서 *누가 판단하는가*의 문제. 본 DP에서 다룰지 후속 DP로 넘길지 미정.
