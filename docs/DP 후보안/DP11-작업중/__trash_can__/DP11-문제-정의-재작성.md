# DP11 문제 정의 (재작성 초안)

---

## 상황

Orchestrator는 Intent를 받아 실행하는 동안, 새로운 Intent가 계속 들어올 수 있습니다.

```
"토요일 저녁 식사 예약해줘"          ← Intent A 시작, 식당 검색 중
       ... 10초 후 ...
"아이도 갈 수 있는 곳으로"           ← Intent B 도착 (A에 조건 추가? 별개?)
       ... 5초 후 ...
"내일 날씨도 알려줘"                ← Intent C 도착 (A와 무관)
       ... 30초 후 ...
"아 그냥 일요일 점심으로 바꿔"       ← Intent D 도착 (A의 시간 조건 대체)
```

이 시점에서 시스템은 이미 식당 후보를 검색해 두었을 수 있습니다.

---

## 풀어야 하는 문제

새로운 Intent가 들어올 때마다, 시스템은 두 가지 판단을 해야 합니다.

### 질문 1. 이 Intent는 기존 활성 Intent와 어떤 관계인가?

새 Intent가 들어오면, **현재 살아있는 모든 Intent**와의 관계를 판단해야 합니다.

| 관계 | 예시 |
|---|---|
| **병합** | B("아이 동반")는 A("식사 예약")의 조건을 보충 → A에 병합 |
| **대체** | D("일요일 점심")는 A의 시간 조건을 덮어씀 → A의 일부를 대체 |
| **차단** | 캘린더 충돌 감지 → A의 실행을 차단 |
| **취소** | "식사 예약 취소해줘" → A 전체를 취소 |
| **무관** | C("내일 날씨")는 A와 아무 관계 없음 → 독립 |

이 판단은 Intent가 들어온 **시점과 무관하게**, 대상 Intent가 활성 상태인 한 계속 가능해야 합니다. 500ms 안에 들어왔든 2분 뒤에 들어왔든, A가 살아있으면 관계 판단이 일어납니다.

### 질문 2. 관계가 없는 Intent들은 동시에 실행할 수 있는가?

위 예시에서 A("식사 예약")와 C("날씨 알려줘")는 서로 충돌하지 않습니다.
이 경우:

- **순차 처리:** A가 끝날 때까지 C는 대기 → 안전하지만 느림
- **동시 처리:** A와 C를 병렬로 실행 → 빠르지만 자원 경합·상태 꼬임 가능성

동시 처리를 허용한다면, **"충돌 없음"을 어떻게 판단할 것인가**가 뒤따릅니다.

| 충돌 판단 기준 후보 | 설명 |
|---|---|
| 같은 Resource를 건드리는가 | 캘린더, 예약 API 등 대상이 겹치면 충돌 |
| 같은 목표(goal)에 속하는가 | "식사 예약" 관련이면 충돌, "날씨"는 독립 |
| 같은 디바이스 화면을 점유하는가 | UI 제어가 동시에 필요하면 충돌 |

---

## 이 문제가 단순하지 않은 이유

두 질문이 서로 얽혀 있기 때문입니다.

```
질문 1 (관계 판단) ──→ 병합/대체/취소/무관 결정
                            │
                            ▼
질문 2 (동시 실행 여부) ──→ 관계 판단 결과에 따라 동시 실행 후보 조합이 달라짐
                            │
                            ▼
                     다시 질문 1로: 동시 실행으로 만들어진 활성 Intent 집합 위에서
                                    다음에 들어오는 Intent의 관계 판단이 다시 일어남
```

또한 이 판단은 **한 번으로 끝나지 않습니다.**
Intent A가 살아있는 동안 Intent B, C, D, E가 순차적으로 들어올 수 있고, 매번 위 두 질문을 다시 풀어야 합니다.

---

## 정리: Decision 포인트

이 DP에서 결정해야 하는 것은:

> **활성 Intent가 존재하는 동안 새로운 Intent가 들어왔을 때, 관계 판단·동시 실행 여부를 어떤 구조로 처리할 것인가?**
