nfr 추가/개선 (운영 기준 — 분산 일관성 반영)

> **반영 배경:** N-party Communication에서 일부 단말 결함·네트워크 단절 시 사본 간 상태가 어긋나면, 사용자가 위임한 자율 행위(예약·결제·약속)가 *동일 협상에 대해 단말마다 다른 결과로 인지*되는 사고가 발생한다. PoC 기준선(95%)이 아니라 **사용자가 대리인에 위임 가능한 운영 신뢰 수준**으로 임계값을 산정한다 — 금융 결제 시스템 정합성(99.99%+), 예약 시스템 이중예약률(0.01% 이하), 자율 대리 행위의 직관적 한계(100건 중 1건 이하)를 참조한다.
>
> **Reliability 속성 정의 보강 (ISO/IEC 25010:2023):** 다자간(N-party) Communication 환경에서는 단일 인스턴스 단위가 아닌 *시스템 전체 일관성* 관점에서 두 부특성을 평가한다.
> - **Fault Tolerance**: 부분 결함 시에도 시스템 전체가 무한 대기·일관성 깨짐 없이 의도된 종료 상태에 도달 (→ Liveness)
> - **Recoverability**: 결함 해소 후 N개 사본이 같은 상태로 수렴 (→ Safety)

### NFR-MAF-01. Reliability - Communication 완결성 (Fault Tolerance)

정의: 시작된 Communication 동작(Negotiation, Collaboration, Knowledge Sharing, Remote Monitoring)은 일시적 오류가 발생하더라도 자동 복구를 통해 정상적으로 완료되어야 한다. 다자간(N-party) Communication에서는 일부 참여자 결함 발생 시에도 시스템 전체가 무한 대기 없이 합의·실패·무효 중 하나의 종료 상태에 도달해야 한다.

측정 기준:

| 항목 | 측정 정의 | 임계값 | 근거 |
|---|---|---|---|
| 일시적 오류 시 자동 재시도 횟수 | - | **최대 3회** | 표준 네트워크 재시도 정책 기준. Exponential backoff 적용(1초 → 2초 → 4초) |
| 3회 재시도 후에도 실패 시 | - | FR-MAF-08에 따라 사용자 에스컬레이션 | - |
| **(N-party) 부분 결함 시 진행 보장** | 정족수 이상 생존 상태에서 장애 발생 ~ 종료 상태(합의/실패/무효) 도달 시간 | **60초 이내 ≥ 99%** | 일반 모바일 앱 응답 성공률 기준선. NFR-MAF-04의 1:1 협상 한계(60초)와 정합 — 장애 후 잔여 진행이 처음부터 시작보다 오래 걸려선 안 됨 |
| **(N-party) 정족수 미달 시 무효 전이** | 정족수 미달 감지 ~ 명시적 "협상 무효" 상태 전이 시간 | **10초 이내 = 100%** | 의사 합의(살아남은 일부만의 합의) 방지는 Safety 절대 조건. 결정론적 타임아웃 로직이라 100% 달성 필수. 다른 상태 전이 작업(Discovery 5초)과 정합 |

상세 설명:
* 완결성 판단 범위: Action(캘린더 등록, 예약 API 호출 등) 단계까지 포함하여 완결로 간주한다.
* 협상 결렬(양측 조건 불일치로 합의 불가 판정)은 기술적 실패가 아니며 완결성 위반으로 간주하지 않는다.
* **N-party 정족수 기준**: 단순 과반(N/2 + 1)을 적용한다. 협상 시작 시점에 참여자 수 N이 확정되며, 협상 중 합류·이탈은 본 PoC 범위에서 제외한다.
* **"무한 대기" 판정 기준**: 60초 이내에 종료 상태에 도달하지 못한 경우 결함으로 간주한다.
* **"종료 상태"의 정의**: (a) 살아있는 참여자 간 합의 도달, (b) 명시적 협상 실패 통보, (c) 정족수 미달로 인한 명시적 무효 통보 중 하나.
* **Liveness 의의**: 시스템이 *진행 안 함*이 아니라 *명시적으로 끝남*을 보장해야 사용자가 다음 행동(재시도·취소·수동 처리)을 결정할 수 있다.

### NFR-MAF-03. Reliability - 세션 복구 가능성 (Recoverability)

정의: 앱 재시작 또는 일시적 네트워크 장애 후에도 진행 중이던 Communication 세션을 복구할 수 있어야 한다. 다자간(N-party) Communication에서는 복구 후 살아있는 모든 참여자의 사본이 동일한 상태로 수렴해야 한다.

측정 기준:

| 항목 | 측정 정의 | 임계값 | 근거 |
|---|---|---|---|
| 세션 복구 소요 시간 | - | **30초 이내** | 30초 초과 시 사용자가 자발적으로 재시작을 선택하는 임계값 기준 |
| 세션 복구 성공률 | - | **95% 이상** | 단일 단말 복구 PoC 기준선 |
| **(N-party) 복구 후 사본 일관성** | 장애 해소 + T_reconcile 시점에 살아있는 사본 간 상태 불일치 발생 케이스 비율 | **불일치 ≤ 0.1%** (1,000 케이스당 1건) | 사용자가 대리인에 위임 가능한 운영 신뢰 수준. 잘 설계된 CRDT·event-sourcing 구현에서 도달 가능. 5%·1% 수준은 *20~100건 중 1건* 사고로 위임 신뢰가 무너지는 빈도 |
| **(N-party) 사본 수렴 시간 (T_reconcile)** | 장애 해소 ~ 모든 살아있는 사본 수렴까지 시간 | **5초 이내** | NFR-MAF-04 N-party 협상 시간(120초)의 약 4%로 협상 영향 최소화. 사용자 인지 시간 내 수렴되어 불일치가 노출되지 않음. 모바일 RTT + 메시지 처리의 실현 가능 하한선 |

상세 설명:
* 현재 우선 적용 범위: Negotiation 세션. 이후 Collaboration, Knowledge Sharing, Remote Monitoring 순으로 확장 적용한다.
* 케이스별 세션 복구 방식이 상이할 경우, 케이스별 세부 요건을 별도로 정의한다.
* **"사본 일관성"의 정의**: 살아있는 모든 참여자의 협상 상태가 다음 세 가지 중 하나로 동일하게 수렴해야 한다.
  - (a) 동일한 합의 상태 — 합의 도달 케이스
  - (b) 동일한 실패 상태 — 합의 불가능 케이스, 동일 사유로 종료
  - (c) 동일한 진행 상태 — 사건 이전과 같은 라운드/제안 컨텍스트로 협상 재개
* **"살아있는 참여자"의 정의**: NFR-MAF-01의 정족수 판정 기준에 따라 살아있는 것으로 식별된 참여자만 일관성 측정 대상이 된다. 죽은 단말의 사본은 제외한다.
* **불일치 측정 방법**: 장애 주입 → 해소 → T_reconcile(5초) 대기 → 살아있는 모든 단말 상태 스냅샷 → 사본 간 비교. 1,000회 반복 시 불일치 발생 케이스 ≤ 1건.
* **본 NFR과 NFR-MAF-01의 관계**: 본 NFR(Recoverability)은 "복구 후 N개 사본이 같은 상태인가"(**Safety**)를 다루고, NFR-MAF-01(Fault Tolerance)은 "결함 발생 시 진행이 보장되는가"(**Liveness**)를 다룬다. 분산 시스템에서 두 속성은 짝으로 충족되어야 한다.
* **아키텍처적 함의**: 본 임계값(불일치 ≤ 0.1%, 수렴 5초)은 *Coordinator 단일 진실 출처 모델*로는 달성이 어렵다(Coordinator 단말 장애 시 모든 사본 stale). CRDT 기반 event-sourcing 또는 RSM(Raft/Paxos) 모델로의 이행이 사실상 필수 전제가 되며, 이는 별도 DP(분산 협상 상태의 일관성 보장 방식)에서 결정한다.
