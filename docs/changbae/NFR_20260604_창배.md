# 비기능 요구사항


## 1. IDS 비기능 요구사항


### Performance - 실시간성

사용자의 Intent를 빠른 시간 내에 인지해야 한다.


## Performance - 실시간성

IDS의 Intent 인지 상황을 사용자에게 즉각적으로 알려주어야 한다.


### Resource Efficiency

IDS 동작 시 On-Device의 자원(CPU, Memory)을 최소로 사용해야 한다.

 



## 2. MAF 비기능 요구사항


### Reliability? (완결성?)

시작된 Communication 동작(Negotiation, Collaboration, Knowledge Sharing, Remote Monitoring)은 정상적으로 완료되어야 한다. (Action까지 포함)



### Resource Efficiency

On-Device의 CPU, memory 자원을 최소로 사용해야 한다.



### Performance - Latency(시간 반응성)

Communication 동작은 빠른 시간내에 완료되어야 한다.
- 총시간: task가 MAF로 전달된 시점부터 Action 완료까지 걸리는 시간 
- Discovery: Agent 상호 발견 및 연결까지 걸리는 시간
- Negotiation: 협상 시작부터 협상 완료 까지 걸리는 시간
- Collaboration: 협업 시작까지 걸리는 시간
- Knowledge Sharing: oo부터 oo까지 걸리는 시간
- Remote Monitoring: oo부터 oo까지 걸리는 시간


### Performance - Latency(시간 반응성)

Communication 동작 중 사용자의 직접 개입 시 변경 요청이 반영되는데 걸리는 시간을 최소화 해야 한다.

- 소요 시간: 사용자 직접 개입 부터 MAF에서 Intent 변경 인지 후 적용까지 (최종 종료까지 보는 것이 아님)



### Tracability

Communication의 진행 과정을 사용자가 원할 때 확인할 수 있어야 한다.



### 보안 - 개인정보 보안

Agent간 Communication 수행 시 민감 정보 암호화를 하고 개인 정보 공유를 최소화해야 한다.


### 세션 복구 가능성

앱 재시작 또는 일시적 네트워크 장애 후에도 negotiation session을 복구 할 수 있어야 한다.
