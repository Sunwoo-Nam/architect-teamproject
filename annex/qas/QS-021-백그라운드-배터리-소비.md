# QS-021-백그라운드-배터리-소비

## 개요

### Quality Scenario ID
QS-021

### 제목
백그라운드 배터리 소비

### 설명
IDS가 백그라운드에서 지속적으로 기기 활동을 모니터링하는 동안 플랫폼이 소비하는 배터리 전력. 과도한 배터리 소비는 사용자가 플랫폼을 비활성화하게 만들어 BG-1의 핵심 가치인 상시 모니터링을 무력화한다. Android의 배터리 최적화(Doze 모드, App Standby)와 상충하므로, 백그라운드 처리 스케줄링 방식이 아키텍처 결정에 영향을 미친다.

### 품질 속성
성능 — 자원 효율성

## 환경

### 시스템 상태
IDS가 백그라운드에서 활성화되어 메신저 및 캘린더 앱을 모니터링하는 유휴 상태. 협상은 진행되지 않으며, Android Doze 모드와 App Standby 정책이 적용되는 실제 운용 환경

### 초기 조건
- Android 플랫폼에서 플랫폼 프로세스가 백그라운드 서비스로 실행 중
- MessageMonitorUI와 CalendarMonitorUI가 이벤트 대기 상태
- 배터리 충전 상태가 아닌 배터리 방전 상태 (충전기 미연결)
- Android Doze 모드 및 App Standby 기능이 활성화된 상태
- 화면 꺼짐(Screen Off) 상태에서 측정

### 부하 조건
없음 (협상 미진행, 외부 API 호출 없음, 대기 중 상태)

### 관련 컴포넌트
- MessageMonitorUI: 메신저 앱 이벤트 대기 (폴링 또는 이벤트 리스너 방식에 따라 배터리 소비 차이)
- CalendarMonitorUI: 캘린더 앱 이벤트 대기
- LLMGateway: 온디바이스 LLM 모델 로드 여부에 따른 배터리 소비

## 동작

1. **자극원**: 시스템 운영 환경 (시간 경과, 정상 백그라운드 운영)
2. **자극**: 플랫폼이 백그라운드에서 일정 시간 동안 모니터링 상태 유지 (화면 꺼짐, 충전기 미연결)
3. **반응 주체**: Android 배터리 관리자, 플랫폼 백그라운드 서비스
4. **반응 흐름**:
   - 플랫폼 백그라운드 서비스가 Android foreground service 또는 background service로 실행 유지
   - MessageMonitorUI, CalendarMonitorUI가 각 앱의 이벤트를 설정된 스케줄(주기 폴링 또는 push 알림)로 감지
   - Android Doze 모드 진입 시 플랫폼이 제한된 네트워크 접근 및 CPU 사용으로 동작
   - 일정 시간 경과 후 배터리 소비량을 Android 배터리 통계에서 측정

## 측정

### 측정 항목
IDS 활성 상태, 협상 미진행, 대기 중 시나리오에서 시간당 배터리 소비량

### 측정 공식
```
B_idle = delta_battery / delta_time (% / hour)

delta_battery: 측정 시작 시점의 배터리 잔량(%) - 측정 종료 시점의 배터리 잔량(%)
delta_time: 측정 기간 (시간, hour)

측정 방법:
  - Android Debug Bridge: adb shell dumpsys battery 또는 adb shell dumpsys batterystats
  - 측정 조건: 화면 꺼짐, 충전기 미연결, 협상 미진행, 다른 대형 앱 실행 없음
  - 측정 기간: 최소 2시간 연속 대기 상태
  - 비교 측정: 플랫폼 미설치 상태의 기기 기준 배터리 소비량 (B_baseline)

추가 배터리 소비:
  B_delta = B_idle - B_baseline (% / hour)

모니터링 방식별 비교:
  B_polling: 주기적 폴링 방식에서의 시간당 소비
  B_push: 이벤트 기반(push) 방식에서의 시간당 소비
```
