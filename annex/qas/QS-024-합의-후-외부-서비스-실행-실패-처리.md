# QS-024-합의-후-외부-서비스-실행-실패-처리

## 개요

### Quality Scenario ID
QS-024

### 제목
합의 후 외부 서비스 실행 실패 처리

### 설명
협상이 성공적으로 완료되고 사용자가 합의안을 승인했음에도, Sub-Agent가 외부 예약 API를 호출하는 시점에 서비스 장애가 발생하는 경우다. 협상 세션은 합의 상태로 유지되어야 하고, 실행 실패는 별도로 처리되어야 한다. 합의(AgreementDB)와 실행 결과를 분리하지 않으면 재시도 시 협상 전체를 다시 해야 하는 문제가 발생한다.

### 품질 속성
가용성 — 합의 후 실행 실패

## 환경

### 시스템 상태
협상이 완료되어 AgreementDB에 합의안이 저장되고, 사용자가 합의안을 승인한 이후 Sub-Agent가 외부 예약 API를 실행하는 상태. 외부 서비스가 장애 상태

### 초기 조건
- AgreementDB에 합의안이 '승인됨(APPROVED)' 상태로 저장된 상태
- Sub-Agent가 ExternalServiceAPIAdapter를 통해 외부 예약 API 호출 준비 완료
- 대상 외부 서비스(숙박/교통/식당 예약 API)가 장애 또는 일시적 불가 상태
- 합의안에 재시도 권한이 포함된 경우 재시도 가능

### 부하 조건
Fault injection: 외부 서비스 API 호출 시점에 HTTP 5xx 오류 또는 타임아웃 발생

### 관련 컴포넌트
- AgreementDB: 합의 상태를 실행 상태와 분리하여 저장
- ExternalServiceAPIAdapter: 외부 API 호출 실패를 감지하고 재시도 또는 실패 처리
- Sub-Agent: 외부 API 호출 결과를 TaskDB에 기록하고 Orchestrator에 보고
- TaskDB: 실행 결과(성공/실패/재시도 대기) 상태를 합의 상태와 분리하여 저장

## 동작

1. **자극원**: 외부 서비스 제공업체 (서비스 장애)
2. **자극**: Sub-Agent가 합의된 예약을 실행하기 위해 외부 서비스 API를 호출하는 시점에 서비스 장애 발생
3. **반응 주체**: ExternalServiceAPIAdapter, Sub-Agent, AgreementDB, TaskDB
4. **반응 흐름**:
   - Sub-Agent가 ExternalServiceAPIAdapter를 통해 외부 예약 API 호출
   - ExternalServiceAPIAdapter가 외부 API로부터 오류 응답(5xx) 또는 타임아웃을 수신
   - ExternalServiceAPIAdapter가 Sub-Agent에 실패 결과 반환
   - Sub-Agent가 실행 실패 상태를 TaskDB에 기록 (합의 상태는 변경하지 않음)
   - AgreementDB의 합의안 상태는 '승인됨(APPROVED)'으로 유지 (합의와 실행 분리)
   - 재시도 정책에 따라 Sub-Agent가 일정 시간 후 재시도를 예약하거나 즉시 재시도
   - 사용자에게 실행 실패 알림 전달 및 수동 개입 옵션 안내
   - 실행 실패 이벤트와 합의 상태 보존 여부를 AgentExecutionLogDB에 기록

## 측정

### 측정 항목
합의 후 외부 서비스 장애 주입 시 시스템이 합의 상태를 보존한 채 실행 재시도 또는 사용자 안내를 완료하는 비율

### 측정 공식
```
R_graceful_handling = (정상_처리_완료_건수 / 외부_서비스_장애_주입_건수) × 100 (%)

정상_처리_완료 정의: 다음 두 조건을 모두 충족하는 경우
  1. AgreementDB의 합의안 상태가 '승인됨(APPROVED)'으로 유지됨 (합의 데이터 무결성 보존)
  2. 사용자에게 실행 실패 알림 전달 완료 또는 자동 재시도가 TaskDB에 등록됨

합의_상태_보존률 = (합의_상태_유지_건수 / 외부_서비스_장애_주입_건수) × 100 (%)

측정 방법:
  - ExternalServiceAPIAdapter에 fault injection으로 5xx 오류 및 타임아웃을 N=30건 주입
  - AgreementDB에서 합의 상태 변경 여부 확인
  - AgentExecutionLogDB에서 사용자 알림 전달 또는 재시도 등록 여부 확인
  - 각 처리 결과의 성공/실패를 집계

오류 유형별 세분화:
  R_5xx = (5xx_정상처리 / 5xx_주입) × 100 (%)
  R_timeout = (타임아웃_정상처리 / 타임아웃_주입) × 100 (%)
```
