# QS-012-A2A-프로토콜-버전-업데이트-비용

## 개요

### Quality Scenario ID
QS-012

### 제목
A2A 프로토콜 버전 업데이트 비용

### 설명
A2A 프로토콜 메시지 형식이나 통신 방식이 변경되었을 때 수정이 A2AProtocolAdapter 내부로 한정되어야 한다. 프로토콜은 표준화 과정에서 빈번하게 변경될 수 있으므로, NegotiationController 등 비즈니스 로직 컴포넌트가 프로토콜 세부사항에 의존하면 안 된다.

### 품질 속성
변경 용이성 — 프로토콜 변경

## 환경

### 시스템 상태
A2A 프로토콜 표준의 새 버전이 발표되어 메시지 형식 또는 통신 방식 변경이 필요한 상태

### 초기 조건
- 현재 플랫폼이 A2A 프로토콜 특정 버전으로 상대방 PPA와 통신 중
- NegotiationController, ProposalGenerator 등 비즈니스 로직 컴포넌트가 A2AProtocolAdapter를 통해서만 A2A 통신에 접근
- 새 프로토콜 버전의 변경 사항(신규 메시지 타입, 필드 구조 변경, 인증 방식 변경 등)이 명세화된 상태

### 부하 조건
없음 (개발/유지보수 변경 작업 시나리오)

### 관련 컴포넌트
- A2AProtocolAdapter: A2A 프로토콜 메시지 인코딩/디코딩 및 통신 방식을 내부에 캡슐화 (수정 대상)
- NegotiationController: A2AProtocolAdapter를 통해 통신하는 비즈니스 로직 (수정 불필요 여부 확인 대상)
- ProposalGenerator: A2AProtocolAdapter를 통해 제안 전달 (수정 불필요 여부 확인 대상)
- AgreementDB: 합의 데이터 저장 (프로토콜 변경 영향 여부 확인 대상)

## 동작

1. **자극원**: A2A 프로토콜 표준화 기관 (프로토콜 새 버전 배포)
2. **자극**: A2A 프로토콜 메시지 구조 또는 통신 방식 변경으로 인한 업데이트 요구
3. **반응 주체**: A2AProtocolAdapter (수정 대상), 나머지 컴포넌트 (변경 영향 없어야 함)
4. **반응 흐름**:
   - 새 A2A 프로토콜 버전의 변경 명세 분석
   - A2AProtocolAdapter 내부의 메시지 인코딩/디코딩 로직, 프로토콜 핸드셰이크 처리 수정
   - NegotiationController, ProposalGenerator 등 비즈니스 로직 컴포넌트는 수정 없이 동작
   - 업데이트 후 상대방 PPA와의 통신 호환성 테스트 수행
   - 기존 협상 플로우의 정상 동작을 회귀 테스트로 확인

## 측정

### 측정 항목
A2A 프로토콜 메시지 구조 변경 시 A2AProtocolAdapter 외부에서 수정이 필요한 컴포넌트 수

### 측정 공식
```
C_external = count(A2AProtocolAdapter 패키지/모듈 외부에서 코드/설정 수정이 필요한 컴포넌트)

측정 방법:
  - 변경 시나리오: 프로토콜 메시지 필드 추가/제거, 메시지 타입 신규 추가, 인증 방식 변경 각각에 대해 수행
  - git diff로 변경된 파일 추적
  - A2AProtocolAdapter 패키지/모듈 경계 외부에서 변경된 파일 수 집계

C_external = 0이 목표 (A2AProtocolAdapter 내부 변경만으로 업데이트 완료)

추가 측정:
  T_update = 프로토콜 업데이트 작업 완료까지 소요 시간 (개발자-시간, hour)
```
