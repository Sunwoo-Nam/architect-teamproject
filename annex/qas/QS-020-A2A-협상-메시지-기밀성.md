# QS-020-A2A-협상-메시지-기밀성

## 개요

### Quality Scenario ID
QS-020

### 제목
A2A 협상 메시지 기밀성

### 설명
두 PPA 간 교환되는 협상 메시지(제안, 반제안, 합의안 등)는 전송 중 제3자가 내용을 열람하거나 변조할 수 없어야 한다. 협상 내용에는 사용자의 선호, 예산, 일정 등 민감한 조건이 포함되므로, 통신 채널 보안이 A2AProtocolAdapter 설계에서 보장되어야 한다.

### 품질 속성
보안 — 협상 데이터 보안

## 환경

### 시스템 상태
두 PPA 간 A2A 협상 메시지가 네트워크를 통해 송수신되는 상태. 잠재적인 제3자 공격자(도청, 중간자 공격)가 존재하는 네트워크 환경

### 초기 조건
- A2AProtocolAdapter가 상대방 PPA와 통신 채널을 수립한 상태
- 협상 메시지에 사용자의 가격 조건, 일정 선호, 기기 컨텍스트가 포함될 예정
- 네트워크 경로에 잠재적인 도청 또는 변조 위협이 존재

### 부하 조건
제3자 공격 시뮬레이션: 전송 중 메시지 도청 시도, 전송 중 메시지 변조 시도 각각 수행

### 관련 컴포넌트
- A2AProtocolAdapter: 협상 메시지의 암호화 적용 및 메시지 무결성 검증을 담당

## 동작

1. **자극원**: 제3자 공격자 (네트워크 도청 또는 메시지 변조 시도)
2. **자극**: A2A 협상 메시지 전송 중 도청 또는 변조 시도 발생
3. **반응 주체**: A2AProtocolAdapter (암호화 및 무결성 검증)
4. **반응 흐름**:
   - ProposalGenerator가 협상 제안 메시지를 A2AProtocolAdapter에 전달
   - A2AProtocolAdapter가 메시지를 암호화(예: TLS/mTLS)하여 전송
   - 전송 중 제3자가 메시지를 도청하더라도 원문 내용을 해독 불가
   - 전송 중 제3자가 메시지를 변조할 경우, 수신 측 A2AProtocolAdapter가 무결성 검증 실패를 감지
   - 변조된 메시지는 NegotiationController에 전달되지 않고 거부 처리
   - 거부 이벤트가 AgentExecutionLogDB에 기록

## 측정

### 측정 항목
A2A 메시지 전송 경로에서 암호화 적용 여부 및 전송 중 메시지 변조 시도 감지 및 거부 여부

### 측정 공식
```
암호화_적용_여부 = A2AProtocolAdapter가 전송하는 모든 협상 메시지에 암호화가 적용되는지 여부 (O/X)

측정 방법:
  - 네트워크 패킷 캡처 도구(예: tcpdump, Wireshark)로 A2A 통신 패킷 수집
  - 캡처된 패킷의 페이로드가 평문이 아닌 암호화된 형태인지 확인
  - TLS/mTLS 인증서 검증 여부 확인

변조_감지_거부_여부 = 변조된 메시지가 수신 측에서 감지되어 처리가 거부되는지 여부 (O/X)

측정 방법:
  - 중간자 공격(MITM) 시뮬레이션 도구를 사용하여 전송 중 메시지 비트를 변조
  - 수신 측 A2AProtocolAdapter가 변조를 감지하고 메시지를 거부하는지 확인
  - 변조된 메시지가 NegotiationController에 도달하지 않음을 AgentExecutionLogDB에서 확인

변조_거부율 = (감지_거부_건수 / 변조_주입_건수) × 100 (%)
```
