# QS-014-A2A-협상-시뮬레이션-가능-여부

## 개요

### Quality Scenario ID
QS-014

### 제목
A2A 협상 시뮬레이션 가능 여부

### 설명
실제 상대방 기기와 PPA 없이도 테스트 환경에서 A2A 협상 End-to-End 흐름(제안 → 반제안 → 교착 → 합의)을 시뮬레이션하고 검증할 수 있어야 한다. 이 시스템은 PoC이므로 가설 검증을 위한 재현 가능한 테스트가 핵심이다.

### 품질 속성
테스트 용이성 — 협상 시뮬레이션

## 환경

### 시스템 상태
테스트 환경에서 실제 상대방 PPA 없이 협상 시나리오를 시뮬레이션하는 상태

### 초기 조건
- Mock PPA 컴포넌트가 테스트 환경에서 실행 가능한 상태
- NegotiationController, ProposalGenerator, A2AProtocolAdapter가 Mock PPA와 통신 가능하도록 구성
- 시뮬레이션 대상 협상 시나리오(제안 성공, 반제안, 교착, 최종 합의)가 Mock PPA에 프로그래밍된 상태

### 부하 조건
없음 (테스트/검증 환경)

### 관련 컴포넌트
- A2AProtocolAdapter: Mock PPA와의 통신을 실제 PPA와 동일한 인터페이스로 처리
- NegotiationController: Mock PPA를 상대로 협상 흐름 전체를 실행
- ProposalGenerator: Mock PPA의 반응에 따라 제안 및 반제안 생성
- NegotiationSessionDB: 시뮬레이션 중 협상 상태 저장
- AgreementDB: 시뮬레이션 완료 시 합의안 저장

## 동작

1. **자극원**: 개발자/QA 엔지니어 (협상 시나리오 검증 요구)
2. **자극**: Mock PPA를 사용하여 특정 협상 시나리오(제안 → 반제안 → 합의)에 대한 시뮬레이션 테스트 실행 요청
3. **반응 주체**: NegotiationController, ProposalGenerator, A2AProtocolAdapter, Mock PPA
4. **반응 흐름**:
   - 테스트 실행기가 Mock PPA를 초기화하고 시나리오 스크립트(반제안 응답 전략 등)를 설정
   - NegotiationController가 Mock PPA를 실제 PPA와 동일하게 처리하며 협상 세션 시작
   - ProposalGenerator가 초기 제안 생성, A2AProtocolAdapter가 Mock PPA에 전송
   - Mock PPA가 시나리오 스크립트에 따라 반제안 또는 합의 응답 반환
   - 협상 흐름이 제안 → 반제안 → 합의 또는 교착의 전체 경로를 따라 완료
   - 시뮬레이션 결과가 AgreementDB 또는 NegotiationSessionDB에 기록되어 검증 가능

## 측정

### 측정 항목
Mock PPA를 사용한 협상 시뮬레이션 테스트 실행 가능 여부 및 커버리지

### 측정 공식
```
시뮬레이션_가능_여부 = 실제 상대방 PPA 없이 Mock PPA만으로 협상 E2E 테스트 실행 성공 여부 (O/X)

시나리오_커버리지 = (Mock_PPA로_실행_가능한_협상_시나리오_수 / 전체_협상_시나리오_수) × 100 (%)

전체_협상_시나리오:
  1. 첫 제안에서 합의 (1라운드)
  2. 반제안 후 합의 (2라운드)
  3. 다중 반제안 후 합의 (N라운드)
  4. 교착 상태 선언
  5. 합의 후 사용자 승인
  6. 합의 후 사용자 거부

측정 방법:
  - 각 시나리오에 대해 Mock PPA를 설정하고 테스트 실행
  - 테스트 성공 여부: 협상 흐름이 예상 결과(합의 또는 교착)로 완료되고 DB에 올바른 상태가 저장된 경우
```
