# QS-008-권한-범위-초과-행동-차단

## 개요

### Quality Scenario ID
QS-008

### 제목
권한 범위 초과 행동 차단

### 설명
Agent가 사용자 설정 권한 범위(가격 상한, 허용 협상 유형 등)를 초과하는 행동을 시도할 때 AuthorizationChecker가 이를 빠짐없이 차단하는 능력. 권한 범위를 벗어난 합의가 실행되면 사용자 신뢰를 회복할 수 없다. 모든 자율 행동 경로에서 AuthorizationChecker를 반드시 거치는 설계가 요구된다.

### 품질 속성
보안 — 권한 위임 강제

## 환경

### 시스템 상태
Meta Agent 또는 Sub-Agent가 협상 또는 외부 서비스 실행을 위한 행동을 시도하는 상태. AuthorizationChecker가 활성화되어 모든 자율 행동 요청을 검사하는 상태

### 초기 조건
- 사용자가 AuthorizationDB에 권한 범위(가격 상한, 허용 협상 유형, 허용 서비스 목록 등)를 설정 완료
- AuthorizationChecker가 활성화되어 AgentFactory로부터 생성된 모든 Agent의 행동 요청을 가로챔
- Meta Agent 및 Sub-Agent가 협상 또는 서비스 실행 요청을 준비 중

### 부하 조건
Fault injection: 권한 범위를 초과하는 행동 100건을 다양한 초과 유형(가격 초과, 허용되지 않은 서비스 유형, 허용되지 않은 협상 조건 등)으로 주입

### 관련 컴포넌트
- AuthorizationChecker: 모든 자율 행동 요청의 권한 범위 검사 및 차단
- AuthorizationDB: 사용자 설정 권한 범위 저장
- NegotiationController: AuthorizationChecker의 검사를 거쳐 협상 행동 실행
- ExternalServiceAPIAdapter: AuthorizationChecker의 검사를 거쳐 외부 서비스 호출

## 동작

1. **자극원**: Meta Agent 또는 Sub-Agent (권한 범위를 초과하는 행동 시도)
2. **자극**: Agent가 AuthorizationChecker에 권한 범위를 초과하는 협상 조건 또는 서비스 실행을 요청
3. **반응 주체**: AuthorizationChecker
4. **반응 흐름**:
   - Agent가 특정 행동(협상 제안 생성, 합의안 수락, 외부 서비스 호출 등)을 요청하기 위해 AuthorizationChecker에 검사 요청
   - AuthorizationChecker가 AuthorizationDB에서 해당 사용자의 권한 범위 조회
   - 요청된 행동이 권한 범위를 초과하는 경우, AuthorizationChecker가 즉시 차단 응답 반환
   - Agent가 차단 응답을 수신하고 해당 행동을 중단
   - 차단 이벤트가 AgentExecutionLogDB에 기록되고 사용자에게 알림 전달
   - 권한 범위 내 행동의 경우 AuthorizationChecker가 승인하여 행동 실행 허용

## 측정

### 측정 항목
권한 범위 초과 행동 주입 시 AuthorizationChecker의 차단 비율

### 측정 공식
```
R_block = (차단_건수 / 전체_초과_주입_건수) × 100 (%)

전체_초과_주입_건수: 권한 범위를 초과하도록 설계된 행동 요청의 총 수 (N = 100)
차단_건수: AuthorizationChecker가 해당 행동을 차단한 건수

초과 유형별 세분화:
  R_price = (가격_초과_차단 / 가격_초과_주입) × 100 (%)
  R_type = (유형_초과_차단 / 유형_초과_주입) × 100 (%)
  R_scope = (범위_초과_차단 / 범위_초과_주입) × 100 (%)

측정 방법:
  - 각 초과 유형별로 균등하게 분배하여 총 100건 주입
  - AgentExecutionLogDB에서 차단 기록 집계
  - 차단 우회 여부(권한 범위 초과 행동이 실제로 실행된 경우)도 별도 기록
```
