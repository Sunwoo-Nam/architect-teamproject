# QS-009-민감-데이터-서버-LLM-전송-최소화

## 개요

### Quality Scenario ID
QS-009

### 제목
민감 데이터 서버 LLM 전송 최소화

### 설명
사용자의 메시지 내용, 협상 조건, 개인 일정 등 민감한 데이터가 Orchestrator(서버 LLM)로 전송될 때, 원문 개인 식별 정보가 포함되지 않도록 처리해야 한다. 서버 LLM은 Task 분해와 Agent 생성 프롬프팅에만 사용되므로, 개인 데이터는 온디바이스 LLM 처리 범위 내에 머물러야 한다.

### 품질 속성
보안 — 데이터 프라이버시

## 환경

### 시스템 상태
IDS가 복합 Intent를 감지하고 Orchestrator(서버 LLM)에 Task 분해를 요청하는 상태. 요청 페이로드가 네트워크를 통해 서버 LLM에 전달되는 시점

### 초기 조건
- IDS가 메시지 컨텍스트에서 복합 Intent를 감지 완료
- Orchestrator가 서버 LLM에 Task 분해 요청을 전송하려는 상태
- 감지된 Intent에는 원문 메시지, 개인 일정, 연락처 등 개인 식별 정보가 포함되어 있음

### 부하 조건
다양한 Intent 유형(숙박 협상, 일정 조정, 서비스 예약 등)에서 각각 서버 LLM 전송 요청 발생

### 관련 컴포넌트
- LLMGateway: 서버 LLM 전송 요청의 페이로드를 중개하는 단일 진입점
- IntentDetector: Intent 감지 결과에서 개인 식별 정보 추출 및 익명화 처리 담당
- TaskDB: Task 명세에서 개인 식별 정보 분리 저장 관리

## 동작

1. **자극원**: IDS (복합 Intent 감지 후 Orchestrator 요청 준비)
2. **자극**: Orchestrator가 서버 LLM에 Task 분해 요청 페이로드를 전송
3. **반응 주체**: LLMGateway, IntentDetector
4. **반응 흐름**:
   - IntentDetector가 감지된 Intent에서 Task 분해에 필요한 구조적 정보(Intent 유형, 필요한 서비스 카테고리 등)와 개인 식별 정보(이름, 연락처, 메시지 원문, 구체적 일정 내용)를 분리
   - Task 분해 요청 페이로드에는 구조적 정보만 포함하여 LLMGateway를 통해 서버 LLM에 전송
   - 개인 식별 정보는 온디바이스에 보존하며 서버 LLM 전송 대상에서 제외
   - LLMGateway가 서버 LLM으로의 요청 페이로드를 로그에 기록
   - 서버 LLM의 Task 분해 결과를 수신한 후 온디바이스에서 개인 식별 정보를 결합하여 실행

## 측정

### 측정 항목
서버 LLM으로 전송되는 요청 중 원문 개인 식별 정보가 포함된 요청의 비율

### 측정 공식
```
R_pii_leak = (PII_포함_요청_건수 / 전체_서버_LLM_요청_건수) × 100 (%)

PII(개인 식별 정보) 정의: 이름, 연락처, 메시지 원문, 구체적 일정 내용(날짜/시간/장소의 조합), 금융 정보

측정 방법:
  - LLMGateway에서 서버 LLM으로 전송되는 요청 페이로드를 수집
  - 정규식 기반 PII 패턴 매칭 또는 NER(Named Entity Recognition)으로 PII 포함 여부 검사
  - 전체 서버 LLM 요청 중 PII 패턴이 감지된 요청의 비율 계산

PII 유형별 세분화:
  R_name = (이름_포함_요청 / 전체_요청) × 100 (%)
  R_contact = (연락처_포함_요청 / 전체_요청) × 100 (%)
  R_message = (메시지_원문_포함_요청 / 전체_요청) × 100 (%)
  R_schedule = (구체적_일정_포함_요청 / 전체_요청) × 100 (%)
```
