# QS-019-협상-제안-내-개인-기기-정보-최소화

## 개요

### Quality Scenario ID
QS-019

### 제목
협상 제안 내 개인/기기 정보 최소화

### 설명
ProposalGenerator가 DPA로부터 수집한 기기 컨텍스트(세탁기 스케줄, TV 시청 패턴, 냉장고 상태 등)와 사용자의 개인 조건을 상대방 PPA에게 전송하는 협상 제안에 포함할 때, 협상 타결에 직접 필요한 정보만 포함되어야 한다. 불필요하게 많은 생활 패턴 정보가 상대방에게 노출되면 프라이버시 침해가 발생한다.

### 품질 속성
보안 — 협상 데이터 보안

## 환경

### 시스템 상태
ProposalGenerator가 DPA(Device Proxy Interface)로부터 기기 컨텍스트를 수집하고 협상 제안 메시지를 생성하여 A2AProtocolAdapter를 통해 상대방 PPA에 전송하는 상태

### 초기 조건
- DeviceProxyInterface가 다수의 가전 기기 컨텍스트(세탁기 스케줄, TV 시청 패턴, 냉장고 상태 등)를 수집 완료
- ProposalGenerator가 협상 목표와 기기 컨텍스트를 바탕으로 협상 제안을 생성 준비 중
- A2AProtocolAdapter가 상대방 PPA와 통신 채널 수립 완료

### 부하 조건
다양한 협상 도메인(숙박, 교통, 가사 서비스 등)에서 각각 제안 메시지 생성 및 전송 시도

### 관련 컴포넌트
- ProposalGenerator: 기기 컨텍스트에서 협상 타결에 필요한 최소 정보만 추출하여 제안 생성
- A2AProtocolAdapter: 제안 메시지를 상대방 PPA에 전송하는 통신 채널
- DeviceProxyInterface: 기기 컨텍스트를 ProposalGenerator에 제공

## 동작

1. **자극원**: NegotiationController (협상 라운드 진행 중 제안 생성 요청)
2. **자극**: ProposalGenerator가 DPA 컨텍스트를 포함한 협상 제안을 A2AProtocolAdapter를 통해 전송 시도
3. **반응 주체**: ProposalGenerator, A2AProtocolAdapter
4. **반응 흐름**:
   - ProposalGenerator가 DeviceProxyInterface로부터 기기 컨텍스트 데이터 수신
   - ProposalGenerator가 협상 타결에 직접 필요한 정보(예: 가격 범위, 날짜 범위)와 불필요한 정보(기기 식별자, 생활 패턴 원문, 사용자 고유 ID)를 분리
   - 협상 타결에 직접 필요한 정보만을 포함한 제안 메시지 생성
   - A2AProtocolAdapter가 제안 메시지를 상대방 PPA에 전송
   - 전송된 메시지 내용이 감사 로그에 기록

## 측정

### 측정 항목
A2AProtocolAdapter를 통해 전송되는 협상 제안 메시지 중 협상 조건 타결에 불필요한 개인/기기 식별 정보가 포함된 비율

### 측정 공식
```
R_unnecessary_info = (불필요_정보_포함_제안_건수 / 전체_협상_제안_전송_건수) × 100 (%)

불필요한 개인/기기 식별 정보 정의:
  - 기기 고유 식별자(MAC 주소, 기기 ID)
  - 사용자 생활 패턴 원문(세탁 시간 기록, TV 시청 히스토리 등)
  - 협상 조건과 직접 연관되지 않는 사용자 개인 식별 정보(이름, 주소 등)

측정 방법:
  - A2AProtocolAdapter에서 전송 직전 협상 제안 메시지 페이로드를 수집
  - 사전 정의된 불필요 정보 패턴(정규식, 필드명 목록)으로 포함 여부 검사
  - 협상 도메인별(숙박, 교통, 식당, 가사 서비스 등)로 각각 N=20건 수집 후 집계

정보 유형별 세분화:
  R_device_id = (기기_ID_포함_건수 / 전체_건수) × 100 (%)
  R_pattern = (생활패턴_원문_포함_건수 / 전체_건수) × 100 (%)
  R_personal = (개인식별정보_포함_건수 / 전체_건수) × 100 (%)
```
