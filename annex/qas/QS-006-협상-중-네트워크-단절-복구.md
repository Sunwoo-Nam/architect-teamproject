# QS-006-협상-중-네트워크-단절-복구

## 개요

### Quality Scenario ID
QS-006

### 제목
협상 중 네트워크 단절 복구

### 설명
A2A 협상 진행 중 네트워크가 단절되었다가 복구되었을 때, 협상 세션이 마지막 저장 상태에서 데이터 손실 없이 재개되는 능력. NegotiationSessionDB에 저장된 상태를 기반으로 재개해야 한다.

### 품질 속성
신뢰성 — 연결 복구

## 환경

### 시스템 상태
A2A 협상이 N라운드(N ≥ 1)까지 진행된 상태에서 네트워크 단절이 발생한 후 일정 시간 이후 재연결되는 상황

### 초기 조건
- NegotiationController가 상대방 PPA와 협상 세션을 진행 중
- NegotiationSessionDB에 최소 1라운드 이상의 협상 상태가 저장된 상태
- A2AProtocolAdapter가 상대방 PPA와 통신 채널 유지 중
- 네트워크 연결이 정상 상태

### 부하 조건
네트워크 단절 fault injection: 협상 라운드 중간(제안 전송 후 반제안 수신 전)에 네트워크를 의도적으로 단절하고, 일정 시간 경과 후 재연결

### 관련 컴포넌트
- A2AProtocolAdapter: 네트워크 단절을 감지하고 재연결 시도를 처리
- NegotiationController: 네트워크 단절 시 세션 상태를 보존하고 재개 로직 실행
- NegotiationSessionDB: 협상 세션의 마지막 유효 상태를 영속 저장
- LLMGateway: 재개 후 재추론 요청 처리

## 동작

1. **자극원**: 네트워크 환경 (연결 불안정)
2. **자극**: A2A 협상 메시지 송수신 중 네트워크 단절 발생
3. **반응 주체**: A2AProtocolAdapter, NegotiationController, NegotiationSessionDB
4. **반응 흐름**:
   - A2AProtocolAdapter가 상대방 PPA로부터 응답 수신 실패 또는 연결 종료를 감지
   - NegotiationController에 연결 오류를 통보
   - NegotiationController가 현재 협상 상태(마지막 완료된 라운드)를 NegotiationSessionDB에 저장 (아직 저장되지 않은 경우)
   - A2AProtocolAdapter가 재연결 정책에 따라 주기적으로 재연결 시도
   - 네트워크 재연결 성공 시 A2AProtocolAdapter가 NegotiationController에 재연결 이벤트 전달
   - NegotiationController가 NegotiationSessionDB에서 마지막 저장 상태를 로드
   - 중단된 라운드부터 협상을 재개하고 상대방 PPA에 재개 알림 전송

## 측정

### 측정 항목
네트워크 단절 후 재연결 시 세션 재개 성공률 및 재개까지 소요 시간

### 측정 공식
```
R_recovery = (세션_재개_성공_건수 / 네트워크_단절_주입_건수) × 100 (%)

세션_재개_성공 정의: 재연결 후 NegotiationSessionDB에 저장된 마지막 상태에서 데이터 손실 없이 협상이 계속 진행된 경우

T_recovery = T_negotiation_resumed - T_network_reconnected (초)

T_network_reconnected: 네트워크 재연결이 감지된 타임스탬프
T_negotiation_resumed: NegotiationController가 NegotiationSessionDB에서 상태를 로드하고 협상을 재개한 타임스탬프

측정 방법:
  - 협상 라운드 N = 1, 3, 5에서 각각 단절 주입 후 재연결 (각 10회 반복)
  - AgentExecutionLogDB에서 재개 성공 여부 및 데이터 손실 여부 확인
```
