# QS-002-A2A-협상-완료-시간

## 개요

### Quality Scenario ID
QS-002

### 제목
A2A 협상 완료 시간

### 설명
Meta Agent가 상대방 PPA와 A2A 협상을 시작한 시점부터 합의안이 도출되기까지의 전체 시간. 여러 라운드의 네트워크 왕복과 양측 LLM 추론이 포함된다. 협상이 너무 오래 걸리면 사용자 신뢰를 잃고 직접 개입 빈도가 높아진다.

### 품질 속성
성능 — 응답 시간

## 환경

### 시스템 상태
Meta Agent가 활성화되어 있으며, NegotiationController가 협상 세션을 관리하는 상태. 네트워크 연결이 정상적으로 유지되고 있음

### 초기 조건
- Meta Agent가 협상 목표와 권한 범위를 AuthorizationDB에서 로드 완료
- NegotiationController가 협상 세션 초기화 완료
- A2AProtocolAdapter가 상대방 PPA와 통신 채널 수립
- ProposalGenerator가 초기 제안 생성 준비 완료
- NegotiationSessionDB에 세션 상태 저장 활성화

### 부하 조건
단일 협상 세션, 다중 라운드(최소 2라운드 이상의 제안-반제안 왕복) 진행

### 관련 컴포넌트
- NegotiationController: 협상 흐름 제어 및 라운드 관리
- ProposalGenerator: 각 라운드별 제안 및 반제안 생성
- A2AProtocolAdapter: 상대방 PPA와의 A2A 프로토콜 메시지 송수신
- LLMGateway: ProposalGenerator의 LLM 추론 중개
- NegotiationSessionDB: 협상 세션 상태 저장
- AgreementDB: 최종 합의안 저장

## 동작

1. **자극원**: IDS가 협상 가능한 Intent를 감지하고 Meta Agent를 활성화
2. **자극**: NegotiationController가 상대방 PPA에 협상 세션 개시 요청
3. **반응 주체**: NegotiationController, ProposalGenerator, A2AProtocolAdapter
4. **반응 흐름**:
   - NegotiationController가 협상 세션을 시작하고 NegotiationSessionDB에 세션 초기 상태 저장
   - ProposalGenerator가 LLMGateway를 통해 초기 제안(proposal) 생성
   - A2AProtocolAdapter가 초기 제안을 A2A 프로토콜 형식으로 인코딩하여 상대방 PPA에 전송
   - 상대방 PPA로부터 반제안(counter-proposal)을 수신하면 A2AProtocolAdapter가 디코딩하여 NegotiationController에 전달
   - NegotiationController가 반제안을 평가하고 ProposalGenerator에 다음 라운드 제안 생성 요청
   - 위 과정을 합의 조건이 충족되거나 교착 상태가 선언될 때까지 반복
   - 합의 조건 충족 시 NegotiationController가 합의안을 AgreementDB에 저장하고 세션 종료

## 측정

### 측정 항목
NegotiationController가 협상 세션을 시작한 시점부터 AgreementDB에 합의가 저장되는 시점까지의 총 경과 시간 및 라운드 수

### 측정 공식
```
T_negotiation = T_agreement_saved - T_session_start (분)

T_session_start: NegotiationController가 NegotiationSessionDB에 세션을 최초 기록한 타임스탬프
T_agreement_saved: AgreementDB에 최종 합의안이 저장된 타임스탬프

라운드별 세분화:
T_round_N = T_proposal_received_N - T_proposal_sent_N (초)

N: 라운드 번호 (1, 2, 3, ...)
T_proposal_sent_N: N번째 라운드에서 A2AProtocolAdapter가 제안을 전송한 타임스탬프
T_proposal_received_N: N번째 라운드에서 상대방 반제안을 수신한 타임스탬프

집계: 협상 세션별 총 소요 시간(T_negotiation), 라운드별 왕복 시간(T_round_N), 라운드 수(N) 기록
```
