# QS-003-백그라운드-모니터링-메모리-사용량

## 개요

### Quality Scenario ID
QS-003

### 제목
백그라운드 모니터링 메모리 사용량

### 설명
IDS가 백그라운드에서 기기 활동을 지속적으로 모니터링하는 동안 플랫폼이 점유하는 메모리 크기. Android OS는 일정 메모리 이상을 사용하는 백그라운드 프로세스를 강제 종료하므로, 상시 모니터링의 지속성이 이 값에 달려 있다.

### 품질 속성
성능 — 자원 효율성

## 환경

### 시스템 상태
IDS가 백그라운드에서 활성화되어 메신저 및 캘린더 앱을 모니터링하는 대기 상태. 협상은 진행되지 않으며, 사용자의 직접적인 인터랙션도 없는 유휴 모니터링 상태

### 초기 조건
- Android 플랫폼에서 플랫폼 프로세스가 백그라운드 서비스로 실행 중
- MessageMonitorUI와 CalendarMonitorUI가 각각 해당 앱의 이벤트를 대기 중
- LLMGateway는 초기화되어 있으나 추론 요청은 없는 상태
- NegotiationController 비활성 상태
- AgentFactory, ToolFactory 비활성 상태

### 부하 조건
없음 (협상 미진행, 외부 API 호출 없음, 대기 중 상태)

### 관련 컴포넌트
- MessageMonitorUI: 메신저 앱 이벤트 대기 중
- CalendarMonitorUI: 캘린더 앱 이벤트 대기 중
- LLMGateway: 온디바이스 LLM 모델 로드 여부에 따른 메모리 점유
- IntentDetector: 이벤트 수신 대기 상태로 메모리 점유
- DeviceProxyInterface: 기기 상태 모니터링 대기 상태

## 동작

1. **자극원**: 시스템 운영 환경 (시간 경과, 정상 백그라운드 운영)
2. **자극**: 플랫폼이 백그라운드에서 일정 시간 동안 모니터링 상태 유지
3. **반응 주체**: Android 프로세스 관리자, 플랫폼 백그라운드 서비스
4. **반응 흐름**:
   - 플랫폼 백그라운드 서비스가 Android foreground service 또는 background service로 실행 유지
   - MessageMonitorUI, CalendarMonitorUI가 각 앱의 콘텐츠 변경 이벤트를 폴링 또는 리스닝
   - LLMGateway가 온디바이스 LLM 모델의 메모리 로드 상태를 유지
   - 주기적으로 플랫폼 프로세스의 RSS를 측정하여 메모리 추세 기록

## 측정

### 측정 항목
IDS 활성, 협상 미진행, 대기 중 시나리오에서 플랫폼 프로세스가 점유하는 평균 메모리 크기

### 측정 공식
```
M_idle = mean(RSS_samples) (MB)

RSS_samples: 백그라운드 대기 상태에서 일정 주기(예: 1분 간격)로 측정한 플랫폼 프로세스의 RSS(Resident Set Size) 샘플 집합

측정 방법:
  - Android Debug Bridge(adb shell dumpsys meminfo <package>) 또는
  - /proc/<pid>/status의 VmRSS 값을 주기적으로 수집
  - 측정 기간: 최소 30분 연속 대기 상태

M_peak: 동일 기간 내 최대 RSS 값
M_idle = mean(RSS_samples) (MB)
```
