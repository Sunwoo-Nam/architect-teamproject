# DP02 PoC — 측정 프로토콜

> 이 문서는 [AGENTS.md](./AGENTS.md)의 전제 위에서 **"무엇을, 어떻게 잰다"** 를 정의한다.
> 테스트셋의 구조는 [01-테스트셋-스키마](./01-테스트셋-스키마.md)가 다룬다.

---

## A1. 실험 목록

### 실험 B (핵심) — 결함 주입 누설 측정

- **목적:** DP02의 중심 주장 — *"방안 1은 필터 신뢰에 의존 / 방안 2는 원본 미도달"* — 을 실측한다.
- **방법:** 방안 1의 필터를 **현실적으로 불완전하게** 두고(아래 결함 클래스), 각 시나리오·각 방안에서 LLM-cp로 건너간 모든 텍스트를 누설 오라클(A3)로 채점한다.
- **비교:** 동일 시나리오·동일 결함 모델에서 **방안 1 유출 건수 vs 방안 2 유출 건수.**
- **필터 결함 클래스 (false-negative 유형):**
  - 패러프레이즈 — 사실을 자연어로 환언("세탁 끝나서" = 19:30 노출)
  - 비정형 위치 — 스키마 밖 자유 텍스트 필드에 원본 삽입
  - 부분 노출 — 시간/금액의 일부만 흘림
  - 단위 우회 — "강남역 5분" 같이 정밀도 기준을 비껴감
- **금지:** "필터를 꺼서 샌다"는 측정(AGENTS §2). 결함은 현실적 한계로만 모델링.

### 실험 C — 권한 초과 차단 (FR-MAF-10)

- **목적:** 권한 범위를 초과하는 사실값이 **어디서** 막히는가.
- **방법:** 권한="구 단위"인데 정확 주소, 권한="5만원 이하"인데 정확 금액 등 범위 초과 항목을 주입. 방안 2는 변환(생성) 단계 차단 여부를, 방안 1은 게이트 차단 여부를 본다.
- **지표:** 차단 성공/실패, 차단 위치(생성 vs 게이트).

### 실험 A (보조) — 처리시간·라운드 수

- **명시할 가설:** 실제 LLM 추론시간이 필터의 ms 비용을 압도하므로, 원 문서가 말한 *"라운드당 필터 누적"* 속도 저하 효과는 실제 LLM 환경에서 **작을 수 있다.** 이 가설을 데이터로 확인/반증한다.
- **측정:** 프라이버시 처리 자체 시간(LLM 추론분 제외), 합의까지 라운드 수, 토큰 수.
- **주의:** 지연·배터리는 비대표(AGENTS §5 충실도 한계) → **on-device 실용성 결론 도출 금지.**

## A2. 지표 정의

| 지표 | 정의 | 단위 | 적용 |
|---|---|---|---|
| `leak_count` | 오라클이 유출로 판정한 건수 | 건 | B |
| `leak_rate` | 유출 건수 / 총 egress 메시지 수 | 비율 | B |
| `rounds_to_agreement` | 합의까지 라운드 수(미합의 시 cap) | 회 | A |
| `agreement_reached` | 합의 도달 여부 | bool | A |
| `utility` | 합의안이 허용 공개 범위 내에서 목표를 충족하는가 | 0~1 또는 등급 | A |
| `blocked_oos_count` | 권한 초과 항목 차단 건수(성공/실패) | 건 | C |
| `block_location` | 차단 위치(생성 / 게이트 / 미차단) | 범주 | C |
| `t_transform` | 세션 시작 변환 처리시간(방안 2) | ms | A |
| `t_filter_per_round` | 라운드당 필터 처리시간(방안 1) | ms | A |
| `vault_mem` | PrivacyVault 토큰맵 메모리(방안 2) | KB | A |
| `tokens_out` / `tokens_in` | egress·prompt 토큰 수 | 토큰 | A |

> `utility`의 구체 채점식은 시나리오 `success_criteria`(01 문서 B1)에 묶어 정의한다.

## A3. 누설 오라클 (3층)

자유 텍스트 egress에서 유출을 세는 방법. 단일 방법은 과소/과대 측정하므로 3층으로 교차한다.

1. **결정적(deterministic) 층** — ground-truth 비밀 집합에 대한 정규화 문자열·정규식 매칭(전화·정확 주소·`19:30`·정확 금액). 재현성 최고, 패러프레이즈는 못 잡음.
2. **LLM-judge 층** — egress 텍스트 + 비밀 집합을 주고 *"원본 사실이 직접 또는 패러프레이즈로 드러나는가"* 판정. judge 모델·프롬프트 고정, **LLM-cp와 분리된 별도 인스턴스.** 분산 줄이려 다회+다수결.
3. **수동 표본 층** — 일부를 사람이 점검해 1·2층의 FP/FN을 추정·보고.

- **판정 규칙:** 1층 OR 2층이 양성이면 유출로 카운트.
- **한계 보고:** judge 자체 오류율을 3층 표본으로 추정해 함께 보고한다(오라클 신뢰도 명시).

## A4. 공정성 규칙 (고정 변수)

- 동일 시나리오 · 동일 LLM-on/LLM-cp 모델 · 동일 시드 · temperature=0 · 동일 상대 거동.
- **유일한 독립변수** = 프라이버시 처리 방식(방안 1/2). (분리 측정 시 분류기 모드 realistic/perfect만 추가로 변동)
- 두 방안의 공통 코드는 공유하고, 차이나는 부분만 분기한다.
- 라벨은 정답지로만 쓰고 시스템 입력으로 넣지 않는다(AGENTS §2).
