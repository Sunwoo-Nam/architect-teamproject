## 1.1 배경 — 업계 동향 (2026년 5월 기준)

### 1.1.1 Agentic AI가 시장의 주류로 진입함

2026년의 AI 산업은 **"Tell me" 시대에서 "Do it for me" 시대로** 넘어가는 변곡점에 와 있음. 단순 답변 생성이 아니라, AI가 **사용자를 대신해 실제 행동을 수행**하는 Agentic AI가 시장 주류로 진입한 해임.

| 지표 | 출처 |
|---|---|
| 2026년 말까지 **40%의 엔터프라이즈 앱**이 task-specific AI agent를 탑재 (2025년 5% 미만에서 8배 증가) | Gartner [1] |
| Agentic AI 시장 규모: **$7.6B(2025) → $10.8B(2026)**, 2035년 best-case $450B+ | Gartner [2] |
| 2028년까지 B2B 구매의 **90%가 AI agent 매개**, $15T 이상의 B2B 거래가 agent exchange 통과 | Gartner [3] |
| 개인 AI 비서 시장 2026년 **$4.84B**, CAGR 42.2% | Arahi AI [4] |

이 흐름의 핵심은 "reactive chatbot → proactive·agentic assistant"로의 전환임. 사용자가 묻기 전에 백그라운드에서 모니터링하다가, 트리거가 감지되면 능동적으로 작업을 수행하는 모델이 표준이 되어 가고 있음.

### 1.1.2 스마트폰 OS 자체가 "디지털 워커"로 재정의되는 중

Google·Apple·Samsung 모두 차세대 비서를 **OS 차원의 워크플로 오케스트레이터**로 재정의하고 있음. 단말이 수동적 인터페이스가 아니라 **사용자를 대신해 일을 처리하는 디지털 워커**로 진화함.

- **Google**: Gmail·Photos·YouTube·Search의 컨텍스트를 통합하는 *Gemini Personal Intelligence* 출시 [4]
- **Apple**: iOS 26.4 *Siri 오버홀* — on-screen awareness + cross-app actions [4]
- **OpenAI**: AI agent가 앱을 대체하는 전용 단말 개발 루머 (TechCrunch 2026.4.27) [5]
- **Lenovo**: CES 2026에서 *Personal AI Super Agent* 발표 — Personalized·Perceptive·Proactive 3대 축 [6]

스마트폰이 **"앱의 모음"이 아니라 "Agent의 집합"** 이 되는 패러다임 전환임.

### 1.1.3 Agent 간 협상의 표준 — A2A 프로토콜의 성숙

단말이 *"Agent의 집합"* 으로 진화하면, 다음 과제는 **그 Agent들이 다른 사용자·다른 회사의 Agent와 어떻게 상호작용할지**임. 이 영역의 표준 프로토콜이 빠르게 자리잡고 있음.

- **A2A (Agent2Agent) Protocol**: Google이 2025년 4월 발표 → 1년 만인 **2026년 4월 기준 150+ 조직** 채택. Google·Microsoft·AWS 모든 주요 클라우드에 통합 완료 [12][13]
- 핵심 능력: **distributed cognition** — Agent들이 동적으로 capability를 발견하고, 책임을 협상하며, 자기조직화하여 복잡한 task를 풂 [13]
- 기술 기반: JSON-RPC 2.0 over HTTPS, Agent Card(capability 광고), 동기·스트리밍·비동기 푸시 모두 지원 [14]
- 벌써 공급망·금융·보험·IT 운영 등에서 production 단계로 이동함 [13]

A2A는 **B2B·엔터프라이즈에서 검증되었지만**, "두 사용자의 개인 Agent가 서로 협상하는" **개인 간(Peer-to-Peer) 시나리오에 적용된 사례는 아직 없음**. 이 영역이 비어 있다는 것이 중요함.

### 1.1.4 당사의 현재 위치

이 흐름에서 당사는 **이미 적극적인 플레이어**로 자리잡았음.

- **Bixby 4.0** (2026년 3월 31일 출시): LLM 기반 Agentic 아키텍처로 완전 재설계됨. 명령어 매칭 방식을 버리고, **LLM 코어가 의도를 해석하고 실행 계획을 동적으로 생성**하며, 기기 기능을 callable agent로 호출하는 구조임 [7][8]
- **3억 대 Galaxy 디바이스**에 즉시 배포되었으며, 2026년 내 **8억 대 Galaxy AI 기기** 도달이 목표임 [9]
- **Galaxy S26**: 온디바이스 LLM 탑재 예정 — Compact / Balanced / Supreme 3종의 모델 변종으로 자원 제약에 따라 선택 가능 [10]
- **SmartThings 통합**으로 가전 제어까지 확장됨 [8]
- **Samsung SDS**도 CES 2026에서 AI Agent 기반 업무 변혁 비전을 공개함 [11]

즉, 당사은 **"Bixby라는 단일 비서"** 단계를 이미 통과했고, SmartThings 기반의 내부 기기 생태계 협력도 자리잡아 가는 중임. 그러나 **1.1.3에서 본 P2P/D2D 자율 협상 표준은 아직 적용되지 않은 영역**임 — *"단일 비서"* 에서 *"다른 사용자·다른 회사의 Agent와 자율 협상하는 구조"* 로 진화하는 다음 단계가 본 과제의 자리임.

## 1.2 이 과제가 왜 반드시 필요한가

### 1.2.1 사용자 측면 — 인지 부하의 한계점

현재 AI 비서는 **사용자가 명령했을 때만** 동작함. 하지만 일상의 의사결정은 그 형태가 아님.

- 일정 조율 한 건을 위해 사용자는 **메시지·캘린더·예약 사이를 평균 5~10번 왕복**함
- 가족과 외식 장소 정하기, 친구와 여행 일정 잡기, 가전 A/S 예약 같은 **미세 의사결정**이 하루에 수십 건씩 누적됨
- 이 작업은 인지 부하만 큰 게 아니라, **결정의 질도 떨어뜨림** — 시간이 없으니 자주 가는 곳을 또 고르고, 좋은 옵션을 놓침

기존 비서는 *"내가 명령하면 한 단계를 해 주는"* 도구임. 사용자에게 **진짜 필요한 것**은 *"내가 신경 쓰지 않아도 알아서 상대와 합의해서 결과만 가져다 주는"* Agent임. 이 격차를 메우지 못하면, 사용자에게 AI는 여전히 **"또 하나의 앱"** 에 머무름.

또한 사용자는 점점 더 **프라이버시에 민감**해지고 있음. 클라우드로 모든 메시지·일정이 흘러가는 모델은 거부감이 커지는 중임. **개인 기기 안에서 처리되는 자율 비서**가 아니면 사용자가 마음 놓고 권한을 위임할 수 없음.

### 1.2.2 당사 측면 — 단말 사업의 다음 정체성

당사에게 본 과제는 **"좋으면 하는 것"** 이 아니라 **"안 하면 안 되는 것"** 임. 네 가지 압력이 동시에 작용함.

**(1) 단말의 차별화 변수가 H/W에서 Agent로 이동하는 중**

- **현재:** 스마트폰 H/W 사양 경쟁은 사실상 평준화되어 카메라·디스플레이만으로는 차별화가 어려움. Bixby 4.0(2026.3.31 출시) [7]으로 단일 사용자 안에서의 비서 단계는 통과했으나, **다른 사람의 Agent와 협상하는 영역**은 여전히 비어 있음.
- **본 과제로:** P2P 협상 영역(NPN)을 메워 차별화의 다음 축인 *"내 단말이 다른 사람의 단말과 알아서 합의하는 능력"* 을 선점함.

**(2) 당사의 강점인 "기기 생태계"의 자산화 미진**

- **현재:** Google·Apple·OpenAI는 단말 1개 또는 클라우드 중심이지만, 당사는 **스마트폰 + 가전(세탁기·TV·냉장고·에어컨) 전체를 보유**하는 거의 유일한 사업자임. 그러나 가전이 "스마트홈" 카테고리에 머무는 한 이 자산은 Agent Ecosystem의 무형 자산으로 환산되지 않음.
- **본 과제로:** PPA(폰)와 DPA(가전)가 하나의 Agent Ecosystem으로 묶이는 그림을 실증함. 이는 **당사만이 그릴 수 있는 그림**으로, 가전을 *Agent의 협상 근거를 제공하는 컨텍스트 자산*으로 재정의함.

**(3) Agentic AI 표준의 빈자리 — P2P 영역**

- **현재:** A2A 프로토콜은 1년 만에 150+ 조직이 채택했지만, 이는 **B2B·엔터프라이즈** 영역에 한정됨 [13]. 개인 간(Consumer-to-Consumer)·기기 간(Device-to-Device) 자율 협상은 표준이 없음. 이 빈자리는 단순한 기회가 아니라 **두 가지 동시 위험**임:
  - **비전 무력화 위험** — P2P 인터페이스·discovery·negotiation 표준이 없으면 두 사용자의 Agent가 서로 capability를 해석할 수 없어, 1.2.1·1.4의 사용자 가치(시간 회수·결정 품질)가 작동 자체를 못 함. 표준은 본 비전의 *전제 조건*임.
  - **종속화 위험** — A2A는 이미 GCP·Azure·AWS 통합이 끝났음 [13]. P2P 영역도 같은 진영이 먼저 정의하면, 단말 사업자는 *그들의 protocol·schema 위에 얹히는 위치*로 고정됨. H/W 자산을 가진 당사가 OS·클라우드 진영 표준에 종속되는 구조는 단말 차별화 자체를 무너뜨림.
- **본 과제로:** NPN(Neural Proxy Network) 아키텍처를 선제적으로 정의·검증해 **P2P 영역의 표준 설계자** 위치를 확보함.

**(4) 안 하면 어떻게 되는가**

- **현재 위협:**
  - OS 단의 Agent를 보유한 Google·Apple과 단말 차별화 경쟁에서 격차가 벌어짐
  - 가전 사업의 자산이 단순 "스마트홈"에 머물러 차세대 Agent Ecosystem 표준에서 배제됨
  - A2A·MCP 등 표준이 모두 정해진 다음에 들어가면 **표준 추종자** 위치에 고정됨
- **본 과제로:** 위 세 위협을 동시에 차단함.

요약하면, 본 과제는 **"당사가 향후 5년간 단말 사업의 정체성을 결정짓는"** 전략적 PoC임.

## 1.3 과제 목적

### 1.3.1 시스템 목적

> 사용자 권한을 위임받은 AI Agent들이 자율적으로 상호 연결·협상하고, 사용자 Intent로부터 도출된 Sub-Task를 위한 Agent 및 Tool을 동적으로 생성·실행함으로써, **일상의 미세 의사결정에서 발생하는 인지 부하를 줄임.**

### 1.3.2 시스템 비전

> 온디바이스 AI Agent가 사용자를 대신해 다른 사람의 Agent와 자율 협상하고, 스마트 기기 생태계 전반의 Agent들을 조율하는 **Neural Proxy Network(NPN)** 를 실현하여, 개인화된 **기기 중심의 Agent Ecosystem 표준**을 확립함.

## 1.4 기대 효과

본 과제가 성공했을 때 사용자와 당사이 각각 얻는 것은 다음과 같음.

### 1.4.1 사용자가 얻는 것

| 가치 | 구체화 |
|---|---|
| **시간 회수** | 일정 조율·예약·협상에 쓰던 메시지 왕복 시간이 거의 0에 수렴함. 사용자는 결과만 승인하면 됨 |
| **결정 품질 향상** | Agent가 사용자의 과거 선호·일정 충돌·DPA(가전) 컨텍스트를 종합 고려하므로 "급해서 대충 고른 결정"이 줄어듦 |
| **프라이버시 통제권** | 민감 데이터는 온디바이스에서 처리되며, 외부로 나가는 정보는 협상에 꼭 필요한 최소 단위로 제한됨 |
| **최종 통제권 유지** | 사용자는 언제든 협상 중단·조건 수정·결과 거부 가능. Agent가 폭주하지 않음 |
| **고령자·약자에 대한 위임** | 부모 PA ↔ 자녀 PA ↔ Medical SA 같은 시나리오 8 구조로 디지털 약자에게 새로운 가치 제공 |

### 1.4.2 당사가 얻는 것

| 가치 | 구체화 |
|---|---|
| **연구 성과** | 자율 협상·동적 Agent 생성·온디바이스 LLM 실용성을 정량 데이터로 입증한 PoC는 **논문·특허·키노트 발표**의 강력한 자산이 됨 |
| **제품화 옵션** | 본 PoC가 검증되면 Galaxy S27/S28 세대의 OS-level 기능, 또는 Bixby 5.0의 핵심 기능으로 확장 가능함 |
| **Cross-사업부 시너지** | MX(모바일) + DA(가전) + SDS(엔터프라이즈) 사업부를 묶는 **공통 Agent Layer**의 구축 근거 마련. SDS의 엔터프라이즈 A2A 경험과 MX의 컨슈머 사용자 풀이 결합됨 |
